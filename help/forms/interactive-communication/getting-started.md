# インタラクティブ通信（IC）エディターの概要


Adobe Experience Manager（AEM）Formsの&#x200B;**インタラクティブコミュニケーション（IC）エディター**&#x200B;を使用すると、文書、請求書、レターなど、データに基づいたパーソナライズされたコミュニケーションを、デジタルチャネルと印刷チャネルをまたいでデザインおよび配信できます。 このガイドでは、オンボーディングからIC エディターのインターフェイスの操作に至るまで、開始する方法の概要を説明します。


## オンボーディングとアクセス

### アクセス要件

インタラクティブ通信を使用するには、AEM Forms as a Cloud Service環境に&#x200B;**AEM Forms アドオン**&#x200B;が含まれており、アカウントに適切な権限があることを確認してください。

### ブラウザーを確認

サポートされているブラウザーとクライアントプラットフォームについては、リンクされた記事「[&#x200B; サポートされているクライアントプラットフォーム &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-manager-cloud-service/content/overview/supported-platforms)」に従ってください

>[!NOTE]
>
> **迅速なリリースサイクルを備えたブラウザーのサポート：**
> Firefox、Chrome、Edgeのリリースアップデートを定期的に行う。Adobeは、これらのブラウザーの今後のバージョンについて、上記に記載されているサポートレベルを維持することに尽力しています。

### ユーザーの役割と権限の設定

IC Editor機能へのアクセスは、AEM[&#128279;](https://experienceleague.adobe.com/ja/docs/experience-manager-learn/cloud-service/accessing/aem-users-groups-and-permissions)内の ユーザーロールによって管理されます。 インタラクティブ通信の作成と管理に関する主な役割は次のとおりです。

| **役割** | **説明** | **キー権限** |
| --------------------- | ---------------------------------------------------------- | -------------------------------------------- |
| **フォーム作成者** | インタラクティブ通信を作成および編集します。 | ICを作成、編集、プレビュー、公開します。 |
| **テンプレート作成者** | インタラクティブ通信の再利用可能なテンプレートをデザインします。 | テンプレートを作成してロックし、レイアウトを定義します。 |
| **管理者** | ユーザーのアクセス、権限、設定を管理します。 | 役割の割り当て、テンプレートの管理、ICの公開を行います。 |
| **FDM オーサー** | [&#x200B; データ統合用にフォームデータモデル （FDM） &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-manager-cloud-service/content/forms/integrate/use-form-data-model/create-form-data-models)を作成および管理します。 | データソースとモデルを作成、編集、設定します。 |

>[!NOTE]
>
> 対応する機能にアクセスするには、ユーザーが適切なAEM グループ（`forms-user`、`fdm-author`、`template-authors`など）に属していることを確認してください。

## IC エディターへのアクセス

1. **AEM Forms as a Cloud Service** インスタンスにログインします。
2. **Forms > Interactive Communications**&#x200B;に移動します。
3. 「**Create**」→「**Interactive Communication**」をクリックします。
4. **テンプレート**&#x200B;を選択し、データソースを設定し、**作成**&#x200B;をクリックして、**インタラクティブ通信エディター**&#x200B;を開きます。

エディターは、印刷とwebの両方のバージョンのコミュニケーションをデザイン、プレビュー、管理するための統合環境を提供します。

## インターフェイスのナビゲーション

**インタラクティブ通信エディター**&#x200B;のインターフェイスは、すべてのデザインツールと設定オプションに対して作成者が直感的にアクセスできるように設計されています。

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/navigate-the-interface.png)

### &#x200B;1. トップツールバー

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/tool-bar.png)

**場所：**&#x200B;最上位セクション

**目的：**&#x200B;環境へのアクセスとグローバル アクションを提供します。

**次を含む：**

**Adobe Experience Cloud環境** （例：ステージング）と、ユーザー設定と環境アクセスを管理するための&#x200B;**プロジェクトタイトル**、**Beta フィードバック**、**通知**、および&#x200B;**プロファイルコントロール**&#x200B;を表示します。

### &#x200B;2. タブバー（デザイン/マスタータブとファイルコントロール）

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/tab-bar.png)

**場所：**&#x200B;上部ヘッダーの下

**目的：** ビューと通信ファイルを管理します。

**次を含む：**

**タブ：** レイアウトと再利用可能なマスターデザイン用に、**デザインビュー**&#x200B;と&#x200B;**エレメントページビュー**&#x200B;を切り替えます

**ファイル名：**&#x200B;現在の通信のタイトルを表示します（例：ic-11）。

ルール、作成、ズーム（85%）、取り消し/やり直し、削除、PDF プレビュー、保存などの&#x200B;**ビューコントロール：** オプション

### &#x200B;3. 左側のパネル（ナビゲーションツールとコンポーネントツール）

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/left-panel.png)

**場所：** インターフェイスの左側

**目的：** プロジェクト構造、再利用可能なアセット、データ バインディングにアクセスします。

**次を含む：**

* **ホームページ：** ユーザーをメインのインタラクティブ通信（IC）ホーム画面に移動します。この画面では、既存のICとフォルダーを表示および管理できます。

* **メニューパネル：**&#x200B;定規、オブジェクト境界、グリッドにスナップ、フィーチャ オブジェクトにスナップ、XDPの読み込み機能など、表示関連のオプションを表示します。

* **階層表示：**&#x200B;通信のコンポーネント構造を表示し、ページ、パネル、およびサブフォームの組織を示します。

* **コンポーネントライブラリ：** キャンバスにドラッグできるテキスト、画像、サブフォーム、バーコードなどのデザイン要素が含まれています。

* **フラグメント：**&#x200B;複数のコミュニケーションで、定義済みのデザイン ブロックとコンテンツ ブロックを再利用できます。

* **データモデル：**&#x200B;動的データをバインドするために、通信を基になるフォームデータモデル（FDM）に接続します。

### &#x200B;4. 中央Workspace（デザインキャンバス）

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/canvas.png)

**場所：** インターフェイスの中心

**目的：** インタラクティブ通信をデザインするためのプライマリワークスペース。

**機能：**

* ライブラリからのコンポーネントのドラッグ&amp;ドロップ

* ビジュアルレイアウトの配置と書式設定

* ページ、サブフォーム、フィールドの追加または編集

* 左下のコントロールを使用して、ページ間を移動する（例：「1 of 1」）

* 公開前に最終レイアウトをプレビューする

### &#x200B;5. 右パネル（プロパティパネル）

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/right-panel.png)

**場所：**&#x200B;画面の右側

**目的：** コンポーネントの動作とスタイルをカスタマイズします。

**次を含む：**

* 一般設定（名前、種類、流動/配置）

* レイアウトとアピアランスのオプション

* ページネーション、位置、プレゼンスおよびデータバインディングのコントロール
