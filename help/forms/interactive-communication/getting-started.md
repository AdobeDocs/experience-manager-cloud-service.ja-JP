# インタラクティブ通信（IC）エディターの基本を学ぶ

>[!NOTE]
>
> インタラクティブ通信機能は、早期導入プログラムで利用できます。 勤務先のアドレスから `aem-forms-ea@adobe.com` にメールを送信して、アクセスをリクエストします。

>[!IMPORTANT]
>
> **ドキュメントは変更される場合があります**：このプロンプトライブラリは現在製品に対してテスト中であり、更新および改訂される可能性があります。早期導入プログラム中に Forms Experience Builder が進化し続けると、プロンプト、例、ベストプラクティスが変わる可能性があります。

Adobe Experience Manager（AEM）Formsの **インタラクティブ通信（IC）エディター** を使用すると、あらゆるデジタルチャネルと印刷チャネルにわたって、明細書、請求書、レターなど、パーソナライズされたデータドリブン型のコミュニケーションをデザインして配信できます。 このガイドでは、オンボーディングから IC エディターインターフェイスのナビゲートまで、使い始める方法の概要を説明します。


## オンボーディングとアクセス

### アクセス要件

インタラクティブ通信を使用するには、AEM Forms as a Cloud Service環境に **AEM Forms アドオンが含まれていること** およびアカウントに適切な権限が付与されていることを確認してください。

### ブラウザーの検証

サポートされているブラウザーとクライアントプラットフォームを知るには、リンクされている記事 [&#x200B; サポートされているクライアントプラットフォーム &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-manager-cloud-service/content/overview/supported-platforms) に従います

>[!NOTE]
>
> **リリースサイクルの短いブラウザーのサポート：**
> Firefox、Chrome、Edge の各リリースは、定期的にアップデートされます。アドビは、これらのブラウザーの今後のバージョンに対して、上記のサポートレベルを維持するよう尽力します。

### ユーザーの役割と権限の設定

IC Editor 機能へのアクセスは、[AEM内のユーザーの役割 &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-manager-learn/cloud-service/accessing/aem-users-groups-and-permissions) によって制御されます。 インタラクティブ通信の作成と管理において、重要な役割を以下に示します。

| **役割** | **説明** | **主な権限** |
| --------------------- | ---------------------------------------------------------- | -------------------------------------------- |
| **フォーム作成者** | インタラクティブ通信を作成および編集します。 | IC を作成、編集、プレビューおよび公開します。 |
| **テンプレート作成者** | インタラクティブ通信用の再利用可能なテンプレートをデザインします。 | テンプレートの作成とロック、レイアウトの定義 |
| **管理者** | ユーザーのアクセス、権限、設定を管理します。 | 役割の割り当て、テンプレートの管理、IC の公開を行います。 |
| **FDM 作成者** | [&#x200B; データ統合用のフォームデータモデル（FDM）を作成および管理 &#x200B;](https://experienceleague.adobe.com/ja/docs/experience-manager-cloud-service/content/forms/integrate/use-form-data-model/create-form-data-models) ます。 | データソースとモデルを作成、編集および設定します。 |

>[!NOTE]
>
> 対応する機能にアクセスするための適切なAEM グループ（`forms-user`、`fdm-author`、`template-authors` など）にユーザーが属していることを確認します。

## IC Editor へのアクセス

1. **AEM Forms as a Cloud Service** インスタンスにログインします。
2. **Forms/インタラクティブ通信** に移動します。
3. 「**作成**」→「**インタラクティブ通信**」の順にクリックします。
4. **テンプレート** を選択し、データソースを設定して **作成** をクリックして **インタラクティブ通信エディター** を開きます。

エディターは、印刷バージョンと web バージョンの両方のコミュニケーションをデザイン、プレビューおよび管理する統合環境を提供します。

## インターフェイスの操作

**インタラクティブ通信エディター** のインターフェイスを使用すると、作成者はすべてのデザインツールと設定オプションに直感的にアクセスできます。

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/navigate-the-interface.png)

### 1.上部ツールバー

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/tool-bar.png)

**場所：** 最上部のセクション

**目的：** 環境アクセスとグローバルアクションを提供します。

**次を含む：**

ユーザー設定および環境アクセスを管理するための **Adobe Experience Cloud環境** （例：ステージング）と、**プロジェクトタイトル**、**Beta フィードバック**、**通知**、および **プロファイル制御** を表示します。

### &#x200B;2. タブバー（デザイン/マスタータブとファイルコントロール）

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/tab-bar.png)

**場所：** 上部ヘッダーの下

**目的：** ビューおよび通信ファイルを管理します。

**次を含む：**

**タブ：** レイアウトおよび再利用可能な要素のデザインのために、**デザインビュー** と **マスターページビュー** を切り替える

**ファイル名：** 現在の通信のタイトルを表示します（例：ic-11）

**ビューコントロール：** ルール、作成、ズーム（85%）、取り消し/やり直し、削除、PDFプレビュー、保存などのオプション

### 3.左パネル（ナビゲーションおよびコンポーネントツール）

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/left-panel.png)

**場所：** インターフェイスの左側

**目的：** プロジェクト構造、再利用可能なアセットおよびデータバインディングにアクセスします。

**次を含む：**

* **ホームページ：** ユーザーをメインのインタラクティブ通信（IC）のホーム画面に移動させます。この画面で、既存の IC とフォルダーを表示および管理できます。

* **メニューパネル：** ルーラー、オブジェクト境界、グリッドにスナップ、フィーチャ オブジェクトにスナップ、XDP 読み込み機能など、ビュー関連のオプションが表示されます。

* **階層ビュー：** 通信のコンポーネント構造を表示し、ページ、パネル、サブフォームの構成を示します。

* **コンポーネントライブラリ：** キャンバスにドラッグできるテキスト、画像、サブフォーム、バーコードなどのデザイン要素が含まれます。

* **フラグメント：** 複数の通信にわたって、事前定義済みの設計とコンテンツブロックの再利用を有効にします。

* **データモデル：** 動的データをバインドするために、通信を基になるフォームデータモデル（FDM）に接続します。

### 4.中央Workspace（デザインキャンバス）

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/canvas.png)

**位置：** インターフェイスの中心

**目的：インタラクティブ通信** デザインするためのプライマリワークスペース。

**機能：**

* ライブラリからコンポーネントをドラッグ&amp;ドロップする

* ビジュアルレイアウトの配置と書式設定

* ページ、サブフォーム、フィールドの追加または編集

* 左下にあるコントロールを使用してページ間を移動します（「1/1」など）

* 公開前に最終的なレイアウトをプレビューする

### 5.右側のパネル（プロパティパネル）

![IC Docu の検索 &#x200B;](/help/forms/interactive-communication/assets/right-panel.png)

**場所：** 画面の右側

**目的：** コンポーネントの動作とスタイルをカスタマイズします。

**次を含む：**

* 一般設定（名前、タイプ、フローレイアウト/配置済み）

* レイアウトと外観のオプション

* ページネーション、位置、プレゼンスおよびデータバインディングのコントロール
