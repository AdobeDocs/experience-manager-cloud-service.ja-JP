# インタラクティブ通信エディターで UI を関連付け

>[!NOTE]
>
> インタラクティブ通信機能は、早期導入プログラムで利用できます。 勤務先のアドレスから `aem-forms-ea@adobe.com` にメールを送信して、アクセスをリクエストします。

**関連付け UI** は、インタラクティブ通信（IC）エディターの上に構築された、専用のシンプルなインターフェイスです。 現場の担当者やサービスエージェントなど、お客様と向き合うプロフェッショナルを対象として、パーソナライズされ、準拠した正確なコミュニケーションをライブインタラクション中にリアルタイムで生成するように設計されています。

![IC 文書の検索 &#x200B;](/help/forms/interactive-communication/assets/associate-ui-preview.png)

## UI インターフェイスを関連付ける

関連付け UI は、クリーンな 2 つのパネルワークスペースを提供し、迅速で安全な通信生成を可能にします。

### 左パネル：データ入力

- 顧客固有の情報を関連付けて入力または確認します。
- 検証、ヘルパーテキストおよび必須フィールドは、正確な入力の手引きとなります。

### 右側のパネル：リアルタイムプレビュー

- 最終ドキュメントのインスタント プレビューを表示します。
- 関連付けるユーザーがデータを入力すると、が自動的に更新されます。

### ドキュメントの即時生成

- 完成した通信を生成またはダウンロードします。

## ユーザーのペルソナと責務

関連付け UI は、3 つのコア役割によって駆動され、それぞれが異なる責任を持ちます。

### 1.管理者

システム設定、ガバナンス、バックエンド統合、ユーザーアクセスを担当します。

| 担当 | フォーカス |
|---------------|-------|
| システム構成 | コアインフラストラクチャ、ユーザーグループ、フォームデータモデル（FDM）、出力を設定します。 |
| ガバナンスとセキュリティ | ユーザー権限を管理し、システムコンプライアンスを確保します。 |
| 統合管理 | バックエンド統合とライブ顧客データ接続を維持します。 |

### 2.作成者

関連付け UI を使用してインタラクティブ通信をデザインおよび管理します。 ß

| 担当 | フォーカス |
|---------------|-------|
| IC オーサリング&amp;デザイン | レイアウト、ブランディングおよび準拠ドキュメント構造を作成します。 |
| フィールド設定 | データフィールドをマッピングし、編集可能、必須および読み取り専用フィールドを定義します。 |
| 公開とイネーブルメント | IC を公開し、関連アクセス用にリンクを共有します。 |

### 3.関連付け

は関連付け UI を使用して、顧客の支援、情報の更新、準拠したコミュニケーションの生成を行います。


| 担当 | フォーカス |
|---------------|-------|
| データの確認 | 左側の入力パネルを使用して顧客データを入力または検証します。 |
| プレビューと検証 | リアルタイムプレビューパネルを使用して精度を確保します。 |
| 配信 | PDF/メールを生成し、承認済みのチャネルを介して送信します。 |

>[!NOTE]
>
> 関連付けは、**forms-associates** グループの一部である必要があります。

## 動的なユースケース

関連付け UI は、リアルタイムのサービスのニーズを持つ業界で重要な、パーソナライズされたドキュメントの即時生成をサポートします。

| 業種 | 使用例 |
|----------|-------------------|
| **金融サービス** | リアルタイムのローン確認レター、リスクプロファイル概要、アカウント作成を生成します。 |
| **保険** | 即時保険証明カードまたは請求処理要約を作成します。 |
| **ヘルスケア** | 計算されたコピーとスケジュールを使用して、患者の治療計画の要約を作成します。 |
| **公共部門** | その場で、警察の検証報告書、市民サービスの領収書、苦情処理の確認書、およびケースの更新概要を生成します。 |
| **政府** | 福祉スキーム登録用の申請ステータス要約、サービス承認レターおよびリアルタイム通信を作成します。 |

## 関連付け UI ワークフローの有効化

作成者は、以下の手順に従って、関連付け UI アクセス用のインタラクティブ通信（IC）を設定して公開できます。

>[!NOTE]
>
> 関連付けでサポートされるコンポーネント：日付フィールド、数値フィールド、テキストフィールド。

### IC の作成

インタラクティブ通信を設計および設定して、ブランディング、データバインディング、コンプライアンスルールおよび統合が正しく設定されるようにします。

### 関連付け UI の有効化

上部のアクションバーで、「UI を関連付け」オプションを有効にして、IC をアソシエイトドリブンで使用できるようにします。

### コンポーネントで UI を関連付けを有効にします

### 編集可能フィールドの設定

「必須フィールド」セクションで、関連付けるユーザーが編集できるフィールドを有効にします。
正確で制御されたデータ入力を確実に行うための検証を設定します。

### IC の公開

すべての設定を完了したら、安全にアクセスできるようインタラクティブ通信を公開します。

### 発行された IC を関連会社と共有する

公開された IC リンクをアソシエイトに提供します。これにより、アソシエイトは認証を行い、顧客固有の情報を入力し、有効な入力を持つ最終的なコミュニケーションを生成できます。

**関連付け UI** は、構造化コンテンツのオーサリングとリアルタイムの顧客エンゲージメントの間のギャップを埋めます。\
直感的な設計、堅牢なバックエンド設定、厳格なコンプライアンス制御を組み合わせることで、組織は **迅速かつ正確でパーソナライズされたコミュニケーション** を大規模に提供できます。
