# インタラクティブ通信エディターでのUIの関連付け

>[!NOTE]
>
> インタラクティブ通信機能は、アーリーアダプタープログラムで利用できます。 勤務先のアドレスから `aem-forms-ea@adobe.com` にメールを送信して、アクセスをリクエストします。

**アソシエイト UI**&#x200B;は、インタラクティブ通信（IC）エディター上に構築された、専用のシンプルなインターフェイスです。 現場スタッフやサービスエージェントなど、顧客と接する専門家向けに設計されており、ライブでのインタラクション中に、パーソナライズされた、コンプライアンスを遵守した、正確なコミュニケーションをリアルタイムで生成できます。

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/associate-ui-preview.png)

## UI インターフェイスの関連付け

アソシエイト UIは、迅速かつ安全なコミュニケーション生成を可能にする、クリーンな2 パネルのワークスペースを提供します。

### 左側のパネル：データ入力

- 担当者は、顧客固有の情報を入力または確認します。
- 検証、ヘルパーテキスト、必須フィールドが正確な入力の指針になります。

### 右パネル：リアルタイムプレビュー

- 最終文書のプレビューを表示します。
- 関連付けがデータを入力すると、自動的に更新されます。

### ドキュメントをすばやく作成

- 最終版コミュニケーションを生成またはダウンロードします。

## ユーザーペルソナと責任

アソシエイト UIは、3つの主要な役割によって駆動され、それぞれ異なる責任を負います。

### &#x200B;1. 管理者

システムの設定、ガバナンス、バックエンドの統合、ユーザーアクセスを担当します。

| 担当 | フォーカス |
|---------------|-------|
| システム設定 | コアインフラストラクチャ、ユーザーグループ、フォームデータモデル（FDM）、出力を設定します。 |
| ガバナンスとセキュリティ | ユーザーの権限を管理し、システムコンプライアンスを確保します。 |
| 統合管理 | バックエンドの統合と顧客データのライブ接続を維持。 |

### &#x200B;2. 作成者

インタラクティブ通信を設計および管理し、アソシエイト UI用に設定します（アソシエイトビューとオプションのワークフローの有効化を含む）。

| 担当 | フォーカス |
|---------------|-------|
| IC オーサリング&amp;デザイン | レイアウト、ブランディング、準拠ドキュメント構造を作成します。 |
| フィールド設定 | データフィールドをマッピングし、編集可能、必須、および読み取り専用のフィールドを定義します。 |
| Publishing &amp; Enablement | ICを公開し、関連アクセス用にリンクを共有します。 |

### &#x200B;3. 関連付ける

アソシエイト UIを使用して、顧客を支援し、情報を更新し、コンプライアンスに準拠したコミュニケーションを生成します。


| 担当 | フォーカス |
|---------------|-------|
| データの確認 | 左側のエントリーパネルを使用して、顧客データを入力または検証します。 |
| プレビューと検証 | リアルタイムプレビューパネルを使用して正確性を確保します。 |
| 配信 | PDF/電子メールを生成し、承認済みのチャネルを介して送信します。 |

>[!NOTE]
>
> アソシエイトは&#x200B;**forms-associates** グループに含まれている必要があります。 オーサーインスタンスのアソシエイト UIからも送信する作成者の場合は、**workflow-users**&#x200B;にも追加します。

## 動的なユースケース

アソシエイト UIは、リアルタイムのサービスが必要な業界にとって不可欠な、パーソナライズされたドキュメントをすばやく生成するのに役立ちます。

| 業種 | 使用例 |
|----------|-------------------|
| **金融サービス** | リアルタイムのローン確認書、リスクプロファイルの要約、アカウント作成を生成します。 |
| **保険** | 即時のプルーフ・オブ・インシュアランス・カードまたは請求処理の概要を作成します。 |
| **ヘルスケア** | 計算されたコピーとスケジュールで、患者の治療計画の要約を作成します。 |
| **公共機関** | 警察の検証報告書、住民サービスの領収書、苦情確認書、ケースの更新概要を、その場で生成できます。 |
| **政府** | 福祉スキームに登録するための申請ステータスの要約、サービス承認レター、リアルタイムのコミュニケーションを作成できます。 |

## アソシエイト UIの有効化

作成者はアソシエイト UIを有効にし、必要に応じて&#x200B;**インタラクティブ通信設定**&#x200B;で送信のワークフローを設定します。

1. **アソシエイトビューを有効にする** — **アソシエイトプロパティ**&#x200B;で、**アソシエイトビュー編集を有効にする**&#x200B;にチェックを入れ、**変更を適用**&#x200B;をクリックしてドキュメントを保存します。
2. **ワークフローの設定（オプション）** — **ワークフロー**&#x200B;で、**更新のワークフローの設定**&#x200B;をオンにして、ワークフローモデルを選択し、必要に応じて成功メッセージとリダイレクト URLを設定します。
3. **編集可能なフィールドを設定** – 関係者が編集および検証を設定できるフィールドを有効にします。
4. **公開して共有** — ICを公開し、関連付けを共有します。

スクリーンショットと送信/ワークフローの動作（作成者は作成者、公開時はアソシエート）に関する手順については、[&#x200B; インタラクティブ通信のアソシエート UIを有効にして設定](/help/forms/interactive-communication/enable-configure-associate-ui.md)を参照してください。 IC送信からPDFを生成するワークフローを構築するには、[&#x200B; アソシエイト UIの送信ワークフロー – IC PDF出力の生成](/help/forms/interactive-communication/submission-workflow-associate-ui-ic-pdf.md)を参照してください。

**アソシエイト UI**&#x200B;は、構造化されたコンテンツ オーサリングとリアルタイムの顧客エンゲージメントのギャップを埋めます。\
直感的なデザイン、堅牢なバックエンド設定、厳格なコンプライアンス管理を組み合わせることで、企業は&#x200B;**迅速かつ正確でパーソナライズされたコミュニケーション**&#x200B;を大規模に提供できます。
