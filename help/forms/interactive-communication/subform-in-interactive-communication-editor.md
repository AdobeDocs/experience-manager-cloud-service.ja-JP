# インタラクティブ通信エディターのサブフォーム

>[!NOTE]
>
> インタラクティブ通信機能は、アーリーアダプタープログラムで利用できます。 勤務先のアドレスから `aem-forms-ea@adobe.com` にメールを送信して、アクセスをリクエストします。

## &#x200B;1. はじめに

インタラクティブ通信エディターのサブフォームは、フィールド、オブジェクト、コンポーネントを論理セクションにグループ化するために使用されるコンテナオブジェクトです。 レイアウトの管理、オブジェクトの配置の制御、ページをまたいだコンテンツフローの定義に役立ちます。 サブフォームは、構造化され、再利用可能で、レスポンシブなコミュニケーションを作成するために不可欠です。特に、動的コンテンツや繰り返しコンテンツを扱う場合はなおさらです。

サブフォームを使用することで、作成者は一貫性を維持し、ページ化を管理し、セクション全体を構造化データソースにバインドできます。

## &#x200B;2. プロパティ

2.1 フォームデザインのレイアウト

- **固定レイアウト：** オブジェクトは、ページ上の正確な位置を維持します。 正確な配置が必要な静的なデザイン（請求書や公式のレターなど）に最適です。

- **フロー可能なレイアウト：** オブジェクトは、コンテンツの長さと画面サイズに応じて動的に調整されます。 レスポンシブデザインや様々なデータ長を必要とするコミュニケーションに適しています。

2.2 サブフォームの配置

- **Pagination:** ページ間でのサブフォームのブレーク方法を制御します。 サブフォームをまとめることや、その中でページ区切りを許可することをオプションとして使用できます。

- **位置：** サブフォームが親コンテナに対して配置されるか、レイアウト内で自然に流れるかを指定します。

- **アピアランス：** サブフォームの背景、境界線、スタイルを定義して、コンテンツを視覚的に分離します。

- **プレゼンス：** ビジネス ルールまたはデータ値に応じて、表示設定（表示、非表示、条件付き）を設定します。

2.3 データバインディング

- サブフォームは、**フォームデータモデル（FDM）**&#x200B;のノードまたは配列に直接バインドできます。

- バインディングを使用すると、サブフォームは、コレクション内の各レコード（複数のポリシー、トランザクション、アドレスなど）に対して動的に繰り返されます。

- データ構造に基づいて、コンテンツの静的な母集団と動的な母集団の両方をサポートします。

## &#x200B;3. 使用方法

サブフォームは、次の目的で広く使用されています。

- ヘッダー、本文、フッターなどのセクションにドキュメントを分類する。

- 表、項目別の請求書、ポリシーのリストなどの繰り返しコンテンツ。

- ページ化を管理してグループ化されたコンテンツを維持。

- ルールやデータ値にもとづくセクションの条件付き表示。

- 通信タイプに応じたレイアウト制御（固定またはフロー可能）の適用。

作成者は、オブジェクトライブラリからキャンバスにサブフォームをドラッグし、プロパティパネルからレイアウト、位置、バインディングを調整できます。

## &#x200B;4. ベストプラクティス

- **適切なレイアウトの選択：**&#x200B;正確な配置を必要とするフォームには固定レイアウトを使用し、データに基づいた動的なコミュニケーションにはフロー可能なレイアウトを使用します。

- **コレクションのバインドを慎重におこなう：** リストまたは配列の場合、サブフォームをテンプレート行を含むFDM コレクションに直接バインドします。

- **ページネーションの最適化：** サブフォーム内の関連オブジェクトをグループ化して、不自然な改行を防ぎます。

- **条件付きプレゼンスを使用：** データ値に基づいてサブフォームを動的に非表示または表示します。

- **階層を維持：**&#x200B;複雑なレイアウトを管理しやすくするために、論理的にサブフォームをネストします。

- **様々なデータを使用してテスト：** デザインが正しく適応するよう、短いデータセット、長いデータセット、空のデータセットを使用してプレビューします。

サブフォームを効果的に活用することで、よりクリーンなレイアウトを実現し、動的コンテンツをより適切に制御して、データ主導のシナリオと静的シナリオの両方にコミュニケーションをシームレスに適応させることができます。
