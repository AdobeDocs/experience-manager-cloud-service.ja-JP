# インタラクティブ通信エディターのルールエディター




## &#x200B;1. はじめに


インタラクティブ通信エディターのルールエディターを使用すると、作成者はコミュニケーションをインタラクティブでインテリジェントにする動的な動作を定義できます。 ルールは、ユーザーの行動や基礎となるデータにもとづいてフィールドの動作、表示、計算方法を制御し、各コミュニケーションがパーソナライズされたコンテキスト認識を維持します。


シンプルな可視設定から複雑な条件付きロジックまで、作成者はルールによってリアルタイムに適応するエクスペリエンスを提供でき、ユーザビリティ、正確性、エンゲージメントが向上します。


![IC ドキュメントを検索](/help/forms/interactive-communication/assets/rule-creation.png)


## &#x200B;2. プロパティ


2.1 フィールドのプロパティとビヘイビアーの設定


- **入力タイプ：** テキスト、数字、日付、ブール値など、フィールドが受け入れるデータのタイプを定義し、適切な検証と書式設定を行います。


- **表示ルール：**&#x200B;条件に基づいてフィールドを表示、非表示、動的に表示するかどうかを制御します（例：「プロモーションコードが適用された場合のみ割引フィールドを表示する」）。


- **デフォルト値：** フィールドの値（今日の日付、データモデルの顧客名など）を事前に定義して、時間を節約し、精度を向上させます。


2.2計算、検証、ルールロジックの実装


- **フィールドロジック：**&#x200B;請求金額の計算や依存フィールドの自動入力など、計算やフィールドの関係を自動化します。


- **カスタムルール：** ルールエディターを使用して、適格性チェックや階層ベースの価格設定などの複雑なシナリオに対する高度なロジックを作成します。


- **条件付きアクション：** フィールドの有効化/無効化、メッセージの表示、異なるセクションへの分岐など、ユーザー入力またはデータ値に対応するトリガーを定義します。


![IC ドキュメントを検索](/help/forms/interactive-communication/assets/rule-creation1.png)


## &#x200B;3. 使用方法


ルールエディターは、フォームやコミュニケーションをレスポンシブでコンテクストに即したものにするために広く利用されています。


- **動的ディスプレイ：**&#x200B;顧客タイプまたは選択したオプションに基づいて、セクションを非表示または表示します。


- **検証：**&#x200B;形式、範囲、または必須の入力を確認して、エラーを防止します。


- **自動計算：**&#x200B;数式（税金、合計、割引など）を使用して、ユーザータスクを簡素化します。


- **ワークフロー制御：**&#x200B;特定のボタン、フィールド、またはアクションは、前提条件が満たされている場合にのみ有効にします。


- **Personalization:**&#x200B;受信者のプロファイル、環境設定、または適格性の基準に合わせてコミュニケーションを調整します。


## &#x200B;4. ベストプラクティス


- **ルールをシンプルにする：**&#x200B;複雑なロジックをより小さなモジュール化されたルールに分割して、メンテナンスを容易にします。


- **可視性ロジックの優先順位付け：**&#x200B;不要なフィールドを早めに非表示にして、ユーザーエクスペリエンスを効率化します。


- **徹底的にテスト：**&#x200B;複数のデータセットを使用してルールをプレビューし、正確性を確認して競合を回避します。


- **デフォルト値を適切に使用する：**&#x200B;ほとんど変更されないフィールドを事前入力しますが、編集は柔軟に行うことができます。


- **条件付きアクションを活用：** インタラクティブ性を高めるためにそれらを適用しますが、ユーザーに負担をかけないようにします。


- **ドキュメントルール：** ビジネスロジックの記録を保持して、開発者と関係者が明確に理解できるようにします。


入念にルールを設定することで、コンテンツ制作者は、データやユーザーの行動にインテリジェントに対応できるコミュニケーションを構築して、プロセスを合理化し、エラーを減らし、シームレスでパーソナライズされた体験を提供することができます。
