# インタラクティブ通信エディターのチェックボックスコンポーネント


## &#x200B;1. 概要

インタラクティブ通信（IC）エディターのチェックボックスコンポーネントを使用すると、ユーザーは単一または複数のバイナリ選択（yes/no、true/false）を行うことができます。 契約条件、嗜好、同意フィールド、オプトインなどに一般的に使用される、コミュニケーションフォーム内でブール値の入力を素早く取得する方法を提供します。

チェックボックスは、柔軟なスタイル設定、データバインディングオプション、表示ルールをサポートし、インタラクティブで使いやすいフォームのデザインに役立つ、軽量かつ強力なツールです。

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/checkbox.png)

## &#x200B;2. プロパティ

2.1基本フィールド

- **名前：** データモデル、ルール、またはスクリプトでチェックボックスを参照するために使用される一意の識別子。

- **切り替え：** チェックボックスをクリックするとオン/オフに切り替えることができます。 これは、シングルモードまたはグループ化モードで使用できます。

- **キャプション：** チェックボックスの横に表示される説明的なラベル。ユーザーが同意または選択している内容をユーザーに案内します。

- **予約：** インタラクションが行われない場合に、読み取り専用またはフォールバックモードで表示されるオプションのプレースホルダーテキストまたはシンボル。

2.2 ポジション

レイアウト内のチェックボックスの位置を定義します。

- **XおよびY座標：** レイアウトグリッド内のチェックボックスの位置を設定します。

- **高さと幅：** チェックボックスの寸法（mmまたはpx）を定義します。特に、カスタムの表示スタイルまたはアイコンを使用する場合に重要です。

2.3 マージン

レイアウト調整のチェックボックスの周囲に間隔を設定します。

- 上（上）

- 下（下）

- 左

- 右

2.4外観

チェックボックスとそのフレームのビジュアルスタイルを制御します。

- **塗りつぶし：** チェックボックスの背景色（選択または未選択の場合）。

- **線：** チェックボックスの境界線のアウトラインの色。

- **線幅：**&#x200B;境界線の太さ。

- **スタイル：**&#x200B;実線、破線、またはカスタム アウトライン パターン。

- **エッジ：** チェックボックスのコーナースタイルを定義します。デザインの好みに応じて、エッジは丸く、エッジはシャープになります。

2.5 プレゼンス

実行時にチェックボックスを表示する方法とタイミングを指定します。

- **表示可能：**&#x200B;は正常に表示され、スペースを占有します。

- **非表示（空き領域を保持）:** UIからは非表示ですが、レイアウト領域は保持されます。 レイアウト破損のない条件付き表示に便利です。

2.6 データバインディング

チェックボックスで外部または内部のデータソースを操作できるようにします。

**データ バインディングの種類：**

**使用名：** コンポーネントのフィールド名を使用して値をバインドします。

**グローバルデータを使用：**&#x200B;通信全体で共有されているグローバルデータモデルに接続します。

**データバインディングなし：** チェックボックスはスタンドアロンのままであり、データモデルに保存されません。

&#x200B;3. 使用状況

チェックボックスは、次のようなシナリオで一般的に使用されます。

- **同意フィールド：**&#x200B;例：「利用条件に同意します」

- **ユーザー設定：**&#x200B;例：「ニュースレターを購読する」

- **複数選択：**&#x200B;例：「該当するすべてのオプションを選択」

- **フォームの確認：**&#x200B;例：「上記の詳細が正しいことを確認しました」

チェックボックスは、レイアウトグリッドまたはパネル内に配置し、グループ化することで、フォームの整理を改善できます。

&#x200B;4. ベストプラクティス

- わかりやすいキャプションを使用することで、オーディエンスが何を選択しているのかを把握できます。

- バイナリ入力にのみチェックボックスを使用して、混乱を避けます。排他的なオプションにはラジオボタンを使用します。

- クリーンなUIのマージンとレイアウト設定を使用して、適切な間隔を確保します。

- キャプチャした選択肢を保存または処理する必要がある場合に、チェックボックスを意味のあるデータモデル参照にバインドします。

- チェックボックスが事前の入力または条件に依存する場合は、表示ルールを使用します。

インタラクティブ通信エディターのチェックボックスコンポーネントは、バイナリ入力のシンプルで必須のコンポーネントです。 スタイル設定、条件付きプレゼンス、柔軟なデータバインディングのサポートにより、スマートデジタルフォームにおけるインタラクティブ性とユーザー制御を強化する上で重要な役割を果たします。 慎重なラベル付け、一貫したスタイル設定、有意義なデータ統合を実施すれば、チェックボックスを利用して、スムーズで直観的なフォーム体験を実現できます。
