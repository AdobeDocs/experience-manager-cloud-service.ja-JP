# インタラクティブ通信エディターの画像コンポーネント


## &#x200B;1. はじめに

インタラクティブ通信エディターの画像コンポーネントを使用すると、作成者は静止画像を挿入して通信レイアウトを強化できます。 このコンポーネントは、視覚的に魅力的なレイアウトを作成し、ロゴやビジュアルアイコンなどのブランド要素を組み込むために不可欠です。 作成者は、マスターページとデザインビューの両方に配置して、PDFなどのさまざまな出力形式で一貫性のある外観を確保できます。

![IC ドキュメントを検索](/help/forms/interactive-communication/assets/image.1.png)

## &#x200B;2. プロパティ

画像コンポーネントには、外観、雰囲気、動作の設定に役立つプロパティがいくつか用意されています。

2.1. 画像の説明

名前：
画像コンポーネントの一意の識別子として機能し、コンポーネント階層またはルールエディター内で簡単に参照できます。

画像を選択：作成者が複数のソースからロゴ、アイコン、その他の静的ビジュアルなどの画像をアップロードまたは参照できるようにします。


2.2. 位置

説明：レイアウト内の画像の空間配置を制御します。

設定 :

X座標とY座標

高さと幅（mm）

2.3 マージン

テキストボックスの周囲の間隔を定義します。

- 上（上）

- 下（下）

- 左

- 右

2.4. 外観

アピアランス：画像フィールドのビジュアルスタイルを定義し、境界線、フラット、高架などのプリセットを選択し、塗りの色、線幅、コーナースタイル（丸みを帯びたまたはシャープ）でカスタマイズします。

2.5. プレゼンス

説明：実行時の画像コンポーネントの表示を決定します。

- オプション：

   - Visible

   - 非表示（スペースを保持）

## （3）使用方法

画像コンポーネントは、次のような用途に適しています。

- ドキュメントヘッダーへのロゴの挿入

- 請求書または見積の製品画像の表示

- コミュニケーションの視覚的なエンゲージメントの強化

## &#x200B;4. ベストプラクティス

- 共有ライブラリの画像を使用すれば、容易に再利用し、一貫性を維持できます。

- 画像の形状を一定に保ち、伸縮やピクセル化を避けます。

- ドキュメントを高速でレスポンシブに保つために、非常に大きな画像ファイルを使用することは避けてください。

- 画像が必要でない場合は、表示または非表示を条件付きで設定します。

AEM Interactive Communicationの画像コンポーネントは、ブランド化され、パーソナライズされた、視覚的に効果的なコミュニケーションを作成する上で重要な役割を果たします。 プロパティを設定可能にすることで、異なる形式をまたいでデザインの一貫性を維持しながら、ユーザーエクスペリエンスを向上させることができます。
