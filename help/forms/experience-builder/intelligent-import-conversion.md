# インテリジェントな読み込みと変換

>[!NOTE]
>
> Forms Experience Builderは、早期アクセスプログラムで利用できます。 開始する前に、アクセスをリクエストし、許可されていることを確認してください。

Forms Experience Builderのインテリジェントなインポートおよびコンバージョン機能を利用すれば、既存のドキュメント、PDF、画像を、AIを活用した分析と変換を使用して、最新のインタラクティブなデジタルフォームに変換できます。

## サポートされているソース形式

### PDF ドキュメント

**AcroForm PDF:**

- フォームフィールドを使用したインタラクティブなPDF forms
- フォームのようなレイアウトを使用した静的PDF
- 構造化されたコンテンツを使用した複数ページのドキュメント

**XFA ベースのPDF:**

- 従来のXFA （XML Forms Architecture） フォーム
- 高度なレイアウトを備えた複雑なビジネスフォーム
- 政府および企業フォーム

**静的PDF:**

- スキャンした文書とフォーム
- インタラクティブな要素を含まない印刷可能なフォーム
- フォームのような構造を持つドキュメント

### 画像ファイル

**サポートされている形式：**

- PNG、JPG、JPEG、GIF
- 紙のフォームの高解像度スキャン
- デジタルフォームのスクリーンショット
- 手描きのスケッチとワイヤーフレーム

**画質の要件：**

- テキスト認識用の最小300 DPI
- 明確で読みやすいテキストとフォーム要素
- テキストと背景の間の良好なコントラスト
- 適切な方向（回転なし）

### デザインファイル

**Figma デザイン：**

- フォームモックアップとプロトタイプ
- UI/UX デザインファイル
- コンポーネントベースのフォームレイアウト

**その他のデザイン形式：**

- Adobe XD ファイル
- スケッチデザイン
- ワイヤーフレーム文書

## インポートとコンバージョンの方法

### 手順1：読み込み機能へのアクセス

1. Forms Experience Builderを開きます
2. インターフェイスの添付ファイル アイコンをクリックします
3. 「インポートと変換」オプションを選択
4. ソースファイルを選択

### 手順2：文書をアップロードする

**PDF ファイルの場合：**

1. PDF ドキュメントを選択
2. AIが構造を分析するのを待ちます
3. 検出されたフォーム要素の確認
4. コンバージョン設定の確認

**画像ファイルの場合：**

1. 画像をアップロード（PNG、JPGなど）
2. AIがレイアウトとテキストを分析し
3. 検出されたフォームフィールドの確認
4. 必要に応じて調整します

**デザインファイルの場合：**

1. デザインファイルをアップロード
2. AIはフォームコンポーネントを抽出し
3. 変換された要素の確認
4. フォーム構造のカスタマイズ

### ステップ 3：レビューとカスタマイズ

最初のコンバージョン後：

1. **検出されたフィールドを確認**：すべてのフォーム要素が正しく識別されていることを確認してください
2. **フィールドタイプを調整**：フィールドタイプを変更（テキスト、ドロップダウン、チェックボックスなど）
3. **検証を追加**: フィールド検証ルールを設定します
4. **スタイル設定のカスタマイズ**：テーマとブランディングの適用
5. **機能をテスト**: フォームの動作とロジックを検証する

## コンバージョンの例

### PDF フォーム変換

**元のPDF フォーム：**

- 名前、電子メール、電話フィールドを含む静的な連絡先フォーム
- 会社情報課
- コメント領域

**変換されたアダプティブフォーム：**

- 検証付きのインタラクティブテキストフィールド
- 企業規模のドロップダウン
- コメント用の複数行テキスト領域
- 電子メール統合を使用した送信ボタン

### 画像からフォームへの変換

**元の画像：**

- 紙の登録フォームの写真
- チェックボックス付き手書きフォーム
- 情報ごとに複数のセクションを作成する

**変換されたアダプティブフォーム：**

- レイアウトに一致するデジタルテキストフィールド
- インタラクティブなチェックボックスとラジオボタン
- 適切な間隔でセクションを整理
- モバイル対応レスポンシブデザイン

### デザインファイル変換

**元のFigma デザイン：**

- フォームコンポーネントを使用した最新のUI モックアップ
- ブランドのスタイルと色
- 複数のパネルを持つ複雑なレイアウト

**変換されたアダプティブフォーム：**

- デザインのピクセルパーフェクトな再現
- インタラクティブフォーム要素
- デバイスを問わないレスポンシブな対応
- ブランドに即したスタイル設定

## 高度なコンバージョン機能

### インテリジェントフィールド検出

AIは、次のことを自動的に検出し、変換します。

- **テキストフィールド**:1行および複数行のテキストエリア
- **選択フィールド**: ドロップダウン、ラジオボタン、チェックボックス
- **日付フィールド**：適切な形式の日付選択
- **数値フィールド**：検証を含む数値入力
- **ファイルのアップロード**：文書と画像のアップロード領域

### レイアウトの保存

コンバージョンは次の状態を維持します。

- **元の構造**: レイアウトと組織を保持します
- **視覚的な階層**：見出しとセクション区切りを維持
- **間隔と整列**：フォームの間隔を適切に保つ
- **ブランド要素**: ロゴとスタイル設定の要素を保持します

### スマート検証

適切な検証を自動的に追加します。

- **電子メールフィールド**：電子メール形式の検証
- **電話番号**：電話番号の形式の検証
- **必須フィールド**：わかりやすい必須フィールドをマークします
- **日付範囲**：適切な日付制約を設定します

## インポートとコンバージョンのベストプラクティス

### ソースドキュメントの準備

**PDF ファイルの場合：**

- テキストが選択可能であることを確認する（画像だけでなく）
- 静的PDFに高画質スキャンを使用する
- コンテンツを論理セクションで整理
- 明快なフィールドラベルを含める

**画像の場合：**

- 高解像度の画像を使用（300以上のDPI）
- 良好なコントラストと読みやすさを確保
- 回転または歪んだ画像を避ける
- すべてのフォーム要素を画像に含める

**デザインファイルの場合：**

- フォームコンポーネントに一貫した命名規則を使用する
- レイヤーを論理的に整理
- 必要なすべてのフォーム要素を含める
- 高品質で書き出し

### コンバージョン後の最適化

**レビューとテスト：**

- すべてのフォームフィールドと検証をテストする
- モバイルの応答性の検証
- アクセシビリティコンプライアンスの確認
- テスト送信機能

**カスタマイズと機能強化：**

- ビジネスロジックと条件付きルールの追加
- 適切なエラー処理を実装する
- 送信エンドポイントの設定
- ブランドのスタイルとテーマの適用

## コンバージョンの問題のトラブルシューティング

### 一般的な問題

**不正なフィールド検出：**

- PDFのテキストの明瞭度を上げる
- 変換後に手動でフィールドタイプを調整する
- ソースでより記述的なフィールドラベルを使用

**レイアウトの問題：**

- 間隔と整列を手動で調整する
- レスポンシブデザインの原則
- さまざまな画面サイズでテストする

**要素がありません：**

- 見つからないフィールドを手動で追加する
- より優れたソース品質での再インポート
- 段階的なコンバージョン手法の活用
