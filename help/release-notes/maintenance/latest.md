# メンテナンスリリースノート {#maintenance-release-notes}

次の節では、Experience Manager as a Cloud Service の最新のメンテナンスリリースに関する技術リリースノートの概要を説明します。

## リリース 21772 {#21772}

2025年8月6日（PT）に公開されたメンテナンスリリース 21772 の継続的な改善点を以下にまとめます。前回のメンテナンスリリースは、リリース 21706 でした。

2025.8.0 機能のアクティベーションでは、このメンテナンスリリースの機能がすべて提供されます。詳しくは、[Experience Manager リリースロードマップ](https://experienceleague.adobe.com/ja/docs/experience-manager-release-information/aem-release-updates/update-releases-roadmap)を参照してください。

### 新機能  {#new-features-21772}

* SITES-30049:UUID を使用してコンテンツフラグメントの言語コピーを取得するための新しいエンドポイントを追加しました。

### 機能強化 {#enhancements-21772}

* CQ-4358722 :Java 11 と Java 17 の間でロケールコードが異なることによって発生するローカライゼーションの問題を解決しました。
* FORMS-19624：インタラクティブ通信（IC）を有効にしました。 これにより、組織は、構造化されたテンプレートと動的データを組み合わせて、明細書、請求書、通信など、パーソナライズされたオンデマンドのコミュニケーションを提供できるようになります。 Web ベースのテンプレート設計、再利用可能なコンテンツフラグメント、ルール主導のバリエーション、シームレスなデータ統合などの機能により、IC はチャネル間で一貫性のあるスケーラブルな顧客通信を可能にします。
* FORMS-19587、FORMS-17107、FORMS-19591、FORMS-19582、FORMS-20129、FORMS-20002、FORMS-19593、FORMS-20655、FORMS-19583、FORMS-18024、FORMS-19581：アダプティブForms ルールエディターに次の機能強化が行われました。
   * 関数リストの `validate` メソッドで、パネル、フィールド、フォームを検証できるようになりました。
   * ES10 以降の機能と静的読み込みをサポートするために、クライアントサイドのカスタム関数の解析を改善しました。
   * ルールエディターに標準の（OOTB）「レコードのドキュメント（DoR）をダウンロード」ボタンを追加しました。
   * ルール内の動的変数がサポートされるようになりました。
   * カスタムイベントに基づくルールの作成が有効になりました。
   * 繰り返し可能なパネルのルールが、最後のパネルインスタンスだけでなく、正しいコンテキストで実行されるようになりました。
   * クエリパラメーター、UTM パラメーター、ブラウザーパラメーターに基づいてルールをトリガーできるようになりました。
   * EDS （エクスペリエンスデータストア）でフォーム固有のカスタム関数スクリプトがサポートされるようになりました。
   * ルールエディターのサクセスハンドラー内の「移動先」アクションで `EVENT_PAYLOAD` を使用するためのサポートを追加しました。
   * ルールエディターの入力パラメーター内のサポートされている関数呼び出しおよび確保されたルールは、関数呼び出しに必要なパラメーターがない場合は保存されません。
   * ルールエディターの UI で壊れているルールがハイライト表示されている。
* FORMS-18450: reCAPTCHA V2 （非表示の reCAPTCHA を含む）を、アダプティブ Formsで簡単に設定および使用できるようになりました。 設定が 1 か所で管理されるようになり、フォームでスパム保護を有効にするのがより簡単になりました。
* FORMS-18385:XDP からの AFP 生成と、Output サービスを介したAEM Formsでのデータ生成のサポートを追加。
* FORMS-17789：ルールエディターに、レコードのドキュメント（DoR）をダウンロードするための標準のボタンが追加されました。
* FORMS-20313、FORMS-2896: コアコンポーネントベースのフォームで特定の機能を無効にするための `dorExclude` プロパティのサポートを追加しました。
* FORMS-20262: クライアント側で無効な添付ファイル （0 バイト）を処理しました。
* FORMS-18347：見つからないフォームコンテナプロキシコンポーネントに関するアダプティブ Forms エディターのログ機能が改善されました。
* FORMS-16205: コアコンポーネントベースのフォームで、無効なコンポーネントをレコードのドキュメント（DoR）から除外しました。
* FORMS-10836：レコードのドキュメント（DoR）のマスターページプロパティの向きが、右から左の言語に対して変更されました。
* SITES-33025：パスではなく ID を使用して新しい CF エディターを開きます。
* SITES-32741:contentf-fragment ページ参照のトリガー更新を非同期で行います。

### 機能の変更 {#altered-functionality-21772}

* SITES-26344：エンドポイント間で `fragmentId`/`modelId` の検証を統合する – これらの ID が検証され、無効な場合は 400 ステータスコードが返されるようになりました。
* SITES-29598：コンテンツフラグメントモデルの更新時に、フラグメント参照フィールドに追加されたコンテンツフラグメント参照を検証します。

### 既知の問題 {#known-issues-21772}

### 廃止された機能と API {#deprecated-21772}

AEM as a Cloud Service で廃止および削除された機能と API について詳しくは、[廃止および削除された機能と API](/help/release-notes/deprecated-removed-features.md) ドキュメントを参照してください。

### セキュリティ修正 {#security-21772}

AEM as a Cloud Service では、プラットフォームのセキュリティとパフォーマンスの最適化に取り組んでいます。 このメンテナンスリリースでは、特定された 35 個の脆弱性に対処し、堅牢なシステム保護に対する取り組みを強化しています。
