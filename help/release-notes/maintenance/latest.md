# メンテナンスリリースノート {#maintenance-release-notes}

次の節では、Experience Manager as a Cloud Service の最新のメンテナンスリリースに関する技術リリースノートの概要を説明します。

## リリース 20783 {#20783}

2025年5月13日（PT）に公開された、メンテナンスリリース 20783 の継続的な改善点を以下にまとめます。前回のメンテナンスリリースは、リリース 20626 でした。

2025.5.0 機能のアクティベーションでは、このメンテナンスリリースの機能がすべて提供されます。詳しくは、[Experience Manager リリースロードマップ](https://experienceleague.adobe.com/ja/docs/experience-manager-release-information/aem-release-updates/update-releases-roadmap)を参照してください。

### 機能強化 {#enhancements-20783}

* FORMS-18455: AEM Forms コアコンポーネントのアダプティブフォームエディターが強化され、データソースツリー内のフォームで既に使用またはマッピングされているデータオブジェクトの視覚的な指標（ドット）を表示できるようになりました。これは、作成者が利用されたデータ要素を簡単に特定するのに役立つ機能です。
* FORMS-18450：この製品は、reCaptcha V2 ドメインロジックを `AdaptiveFormConfigurationServiceImpl` に移行することで強化されます。 この変更は、設定を一元化することを目的としており、コアコンポーネントで非表示の reCaptcha V2 のサポートを追加することに合致します。
* FORMS-19630:AEM 6.5 クイックスタート uber-jar が更新され、最新のアダプティブ Forms コアコンポーネントパッケージが含まれるようになりました。これにより、クイックスタート環境に最新のアダプティブ Forms機能が反映され、従来のコードが置き換えられます。
* FORMS-19125: コアコンポーネントのアダプティブフォームエディターが強化され、データソースツリーの対応するセクションがフォームキャンバスにドロップされたときに、使用可能なアダプティブフォームフラグメントの自動マッピングをサポートするようになりました。 これにより、生産性向上のための主要な機能が基盤エディターからコアコンポーネントに提供されます。
* FORMS-17887:AEM Formsの Output サービスで、AFP （Advanced Function Presentation）形式のドキュメントを生成できるようになりました。 AFP 通信社を利用した高速大容量印刷環境に対するお客様のニーズに応えます。
* FORMS-15089:AEM Formsでは、公開時にフォームのバージョンを管理する機能を導入しました。これにより、フォームを構成するすべてのフラグメントが、公開済みの特定のバージョンにインライン化（埋め込み）されます。 これにより、公開時に表示されたフォームの完全な自己完結型の表現が保証されます。これは、アーカイブ、法的、コンプライアンスの目的にとって重要になる可能性があります。
* SITES-27775：公開中の参照検索が最適化されました。
* SITES-30885：永続クエリでの JSON 処理を最適化しました。
* SITES-25433：ユニバーサルエディターを使用したEdge Delivery：古いバージョンを比較する際のフルページレンダリングをサポートします。
* SITES-27792: ユニバーサルエディターを使用したEdge Delivery: EDS 設定を完全に CA 設定に移動します。
* SITES-19754：ユニバーサルエディターを使用したEdge Delivery：設定が壊れている場合に表示される、説得力のあるエラーメッセージ。
* SITES-30267：ユニバーサルエディターを使用したEdge Delivery：シート V2 （アーリーアクセス）。
* SITES-30328：ユニバーサルエディターを使用したEdge Delivery:Sidekick サポートからのプレビュー。
* SITES-23499：ユニバーサルエディターを使用したEdge Delivery：ブロックオプションに複数のフィールドを使用できます。
* SITES-29987：コンテンツフラグメントモデルの作成時に `previewUrlPattern` を設定する機能を追加しました。
* SITES-29874：コンテンツフラグメント API で LongTextField 参照のサポートを追加。
* SITES-29601:LongText フィールド経由で参照されるコンテンツフラグメントの検証を追加。
* SITES-24623:GETおよび検索フラグメント API から返される ETag をパッチに使用できるようにします。
* SITES-28557:PATCH コンテンツフラグメントで URL パラメーター `references` を使用できるようにします。
* SITES-5358: [OpenAPI] コンテンツフラグメントを子と共にコピー。
* SITES-29614:GET ワークフローエンドポイント。
* SITES-29615：バッチリクエスト API エンドポイントをリストします。
* SITES-25130：コアコンポーネントの 2.28.0 へのアップグレード
* SITES-10575:「MSM Blueprint Bloomfilter Loader」は、100,000 行を超える読み込みを試みます。
* SITES-26711:RTE テキストフィールドのリンクが、MSM ロールアウト上のライブコピーを指すように更新されない。
* SITES-25976:MSM ロールアウト後にエクスペリエンスフラグメント内のリンクが適応しない。

### 修正された問題 {#fixed-issues-20783}

* ASSETS-50994:AemRequestEventFilter でブロックされた受信トラフィック。
* CQ-4358591:「翻訳プロジェクトを作成」オプションを使用してサイト参照パネルから言語コピーを作成する際に、少数の言語のプロジェクトが見つからない。
* CQ-4359108：人間による翻訳の読み込み/書き出しの使用中に XLIFF 2.0 形式が失敗する。
* CQ-4358722:Java 11 と Java 17 ではロケールコードが異なるので、従来の ISO コードではローカライゼーションが機能しません。
* FORMS-19808：遅延読み込みが有効になっているフラグメントを含む大きなフォームを保存する場合、ドラフトをユーザーが取得することはできません。
* FORMS-19887:XFA フォームのドロップダウンフィールドが、最初に読み取り専用アクセスに設定された場合、HTML5 でフォームをレンダリングしたときに、開く/編集可能ステータスに変えることができません。 このフィールドは読み取り専用のままなので、期待どおりに機能するPDF レンダリングとは異なり、ユーザーのインタラクションを防ぎます。
* FORMS-19651：ルールエディターでは、ボタンクリックがバイナリ条件で使用され、同じボタンがそのルールの「then」ステートメントでも使用される場合、ルールは正しく機能しません。
* FORMS-19628: コアコンポーネントベースのアダプティブForms用に自動生成されたレコードのドキュメント（DoR）で、ルートパネルで「タイトルのリッチテキストを許可」オプションが有効になっている場合、ネストされたパネルのタイトルを DoR から除外すると、ルートパネルのタイトルが正しく非表示にならない。
* FORMS-18977：レコードのドキュメント（DoR）サービスで生成された PDF にドキュメントのタイトルがない。 ドキュメントタイトルはアクセシブルな PDF に必須の属性なので、これが、PDF/UA および WCAG 2.1 アクセシビリティ標準への非準拠につながる可能性があります。
* FORMS-18526：条件内に複数のフィールドを含むルールをフィールド間でコピーすると、これらの条件内の固定フィールド参照が、ルールのコピー先の新しいフィールドに更新されず、元のソースフィールドへの参照を誤って保持する。
* FORMS-19047：アダプティブフォームが変更され、AEM Forms（特に、6.5.22.0）で再公開されると、特定のフォーム要素、特にテキストボックスの翻訳が欠落する場合があります。
* FORMS-19234: AEM Formsの PDF のタイムライン機能。PDFの作成とバージョン管理に関する詳細を表示できます。PDFを「Formsとドキュメント」セクションにアップロードすると機能が停止します。
* FORMS-19373：レプリケーションエージェントが設定されていない環境で、「ゴールデンパブリッシュ」プロセス中にレプリケーションエラーが誤って報告されます。
* FORMS-18196:XDP テンプレートに必要なオプションのフィールドデータがリクエストで空のままになっている場合、`generatePrintedOutput` （または `generatePdfOutput`）同期 HTTP API が、想定される 400 （無効なリクエスト）エラーコードではなく、200 （成功）応答コードを誤って返します。
* FORMS-19336: コアコンポーネントのアダプティブフォームエディター（AF2 エディター）で、データSourceツリー内の検索機能が正しく、または期待どおりに動作せず、特定のデータ要素を簡単に見つけることができません。
* FORMS-19629:JSON スキーマパーサーで無効な結果が生成されるか、お客様から提供された特定の JSON スキーマが誤って解釈されています。 この問題は、フラグメントの自動マッピングなど、正しいスキーマ解析に依存する機能に悪影響を与える可能性があります。
* FORMS-19380: コアコンポーネントのアダプティブFormsのバージョン管理サポートの導入により、他の様々なアセットタイプ（基盤Forms、PDF ファイル、テーマ、FDM など）のバージョン管理機能が意図せず有効になりました。これらのアセットタイプの設計やテストは必要ありません。 この意図しない副作用は調査中です。
* FORMS-17707:AEP プラットフォームの「ステージ」環境に接続するように設定されている場合、AEP（Adobe Experience Platform）コネクタが正しく機能しません。

### セキュリティ修正 {#security-20783}

AEM as a Cloud Service では、プラットフォームのセキュリティとパフォーマンスの最適化に取り組んでいます。 このメンテナンスリリースでは、特定された 19 個の脆弱性に対処し、堅牢なシステム保護に対する取り組みを強化しています。
