# AEMドキュメントのジャーニー {#aem-documentation-journeys}

ドキュメントのジャーニーは、複雑で異なる機能を組み合わせ、ベストプラクティス方式でビジネス目標を解決することで、AEMドキュメント内の構造を説明します。 AEMの初心者を念頭に置いて設計されたジャーニーは、AからZの目標を達成するための概念と機能を紹介します。

## ドキュメントジャーニーとは {#what-is}

ドキュメントジャーニーは、AEMを初めて使用する読者が、事前のトピックやAEMの知識を最小限に抑えながら、ビジネス上の問題を最初から最後まで理解し、解決するのに役立つ説明を提供し、様々な複雑なトピックや機能を結び付けます。

ドキュメントのジャーニーは、AEMに習熟していると仮定して、主に単一の機能およびタスクのドキュメントに焦点を当てるAEMの既存の技術ドキュメントとは異なります。 概念と詳細を既存の技術ドキュメントに委ねることに重点を置き、ドキュメントジャーニーは、AEMを初めて利用する読者に対し、ベストプラクティスを使用して一般的なビジネスニーズやワークフローに対処し、プラットフォームの活用方法を理解する機会を提供します。 この理由から、ドキュメントジャーニーは[AEMの既存の技術ドキュメントの代わりではなく、の補完として機能します。](https://experienceleague.adobe.com/docs/experience-manager-cloud-service.html?lang=ja)

### 物語で学ぶ {#narrative}

AEMには、新しい（時には経験豊富な）ユーザーに圧倒されるような、豊富で強力な機能セットが用意されています。 各ジャーニーを、技術的な要件ではなく明確なビジネス目標で開始することで、ジャーニーは、ビジネス目標を達成するために連携して動作するAEMの概念と機能を、読者を導く物語を伝えます。

ストーリーを語ることで、読者はAEMの様々な部分がどのように連携して問題を解決し、遠近法を維持しているかをより深く理解できます。 これにより、リーダは、特徴木のビジネス目的のフォレストを見ることができる。

### 機能以外の概念に焦点を当てる {#concepts}

ドキュメントのジャーニーでは、技術的な機能に焦点を当てる代わりに、AEMの概念を重視します。 クリックするチェックボックスを気にするよりも、AEMが特定の問題を解決する方法を理解していることがより重要であることを認識し、読者は物語を進み続け、全体的な目標を達成するために複数の重要な概念を結び付ける方法を示します。

ジャーニーを使用すると、クリックする各オプションを気にする代わりに、AEMが問題を解決する方法を読者が確実に把握できます。 読者がより深く掘り下げて、より技術的な詳細や、その他のオプションについて学びたい場合は、ジャーニーのすべての部分を、関連する包括的な技術ドキュメントにリンクさせます。

### ベストプラクティスの方向 {#best-practices}

ドキュメントのジャーニーは、Adobeの最新の調査に基づくベストプラクティスの原則、Adobeサービスによる実証済みの実装経験、お客様のプロジェクトからのフィードバックに基づいて設計されています。

AEMを使用してビジネスケースを解決する方法をAdobeが推奨する方法を知るには、ドキュメントジャーニーを使用します。

## ドキュメントジャーニーの構造 {#structure}

ドキュメントジャーニーは、AEMが一般的なビジネス上の問題を解決する方法を示すベストプラクティスベースの紹介として機能します。 この理由から、各ジャーニーはAEMを初めて使用する読者を対象に設計され、ビジネス上の問題を取り上げ、必要な理論を説明し、AEMによる問題の解決方法を順を追って説明します。 ジャーニーの包括的な特性により、AEMを初めて使用する読者や経験豊富なユーザーにとって役立ちます。

一般的なドキュメントジャーニーは、次の部分で構成されます。

* ジャーニーの目標と対象読者の概要
* ビジネス上の問題の説明
* 問題の解決に必要な理論の説明
* 前提条件と要件
* 対象読者の説明
* 実装の手順

ドキュメントジャーニーの目標は、AEMが単一のビジネス上の問題を解決するために様々な機能やツールを使用する方法の基本を読者に理解してもらうことです。 このため、実装の手順では、最も一般的な使用パターンと最も重要な機能およびオプションを示します。 詳細な設定オプションは、技術ドキュメントにリンクされています。

## ドキュメントジャーニー {#who}

AEMドキュメントジャーニーの主な目標は、AEMを初めて使用する読者が、システムの様々で強力な機能をいくつ組み合わせて使用し、一般的なビジネス上の問題を説明を通じて解決できるかを理解できるようにすることです。

AEMを初めて使用する場合で、特定のビジネス事例を念頭に置いて、AEMでの解決方法の概要を確認したい場合は、ドキュメントジャーニーを使用して作業を開始します。

ただし、様々なタイプのAEMユーザーのニーズや期待事項が異なります。 次に例を示します。

* 管理者
* デベロッパー向け
* コンテンツ作成者
* 翻訳スペシャリスト
* コンテンツアーキテクト

各ジャーニーは、ジャーニーの対象となるオーディエンスの明確な声明から始まります。 誰も真空では働かないので、読者がシステムユーザーやペルソナの入力を支援する必要がある場合、これはジャーニー内で明確に説明されます。

既に利用可能なドキュメントジャーニーが多数あります。 各ジャーニーは物語として設計されているので、AEMのコンテキストでトピックを完全に理解するために、はじめにを読み、完全にお読みください。
