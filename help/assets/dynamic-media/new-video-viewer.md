# Dynamic Mediaの新しいビデオビューア {#new-video-viewer-dynamic-media}

Dynamic Media用の新しいビデオビューアは、Adobe Experience Manager（AEM）で最新のビデオ再生体験を提供します。 オーサリング、プレビュー、Sites環境をまたいで、既存のDynamic Media ワークフローと引き続き連携しながら、一貫性のある拡張可能な視聴エクスペリエンスを提供します。

Dynamic Mediaの既存のビデオビューアは、主要な再生要件をサポートしていますが、最新の分析と統合シナリオに対して限定的な拡張性とイベントレベルの統合を提供します

新しいビデオビューアでは、次の方法で制限に対処します。

* より一貫性のある再生体験の提供
* ビューアの明示的な選択を許可する
* プログラマティック消費のための構造化された再生イベントの出力
* 外部分析および外部システムとの統合をサポート

ビューアは追加オプションとして使用でき、サポートされている場合は明示的に選択する必要があります。 既存のビデオビューアは自動的に置き換えられません。

新しいビデオビューアは、既存の実装を中断することなく、強化された拡張可能なビデオエクスペリエンスを必要とする組織を対象としています。

> **メモ**
>
> 新しいビデオビューアは、限定的な機能です。 [&#x200B; サポートチケット &#x200B;](https://helpx.adobe.com/jp/enterprise/using/support-for-experience-cloud.html)を作成することで、有効にすることができます。


## 新しいビデオビューアの仕組み {#how-it-works}

新しいビデオビューアは次のように機能します。

1. ビデオアセットは、Dynamic Mediaと同期されたフォルダーに取り込まれます。
2. ビデオは、**ビデオ （新規）**&#x200B;を使用して、アセットの詳細ページからプレビューできます。
3. 新しいビデオビューアは、サイトページのオーサリング時に&#x200B;**Dynamic Media** コンポーネントで選択できます。
4. 再生中、ビューアは構造化イベントを親ウィンドウに出力します。
5. オプションのビューア修飾子を使用して、再生動作を制御できます。

## 既存のビデオビューアとの主な違い {#key-differences}

| 領域 | 説明 |
|------|-------------|
| Viewerの可用性 | **ビデオ （新規）**&#x200B;という新しいオプションとして表示されます |
| ビューアの選択 | 明示的に選択する必要があります |
| 拡張機能 | 構造化された再生イベントを出力する |
| 統合 | 既存のDynamic Media ワークフローとの引き続きの連携 |

## 前提条件 {#prerequisites}

新しいビデオビューアを使用する前に、次の前提条件を満たしていることを確認してください。

| 要件 | 説明 |
|------------|-------------|
| Dynamic Media sync | アセットフォルダーはDynamic Mediaと同期する必要があります。 |
| ビデオプロファイル | ビデオプロファイルをフォルダーに適用する必要があります。 |
| ビデオアセット | ビデオはフォルダーに取り込む必要があります。 |

新しいビデオビューアは、**AEM as a Cloud Service バージョン 2025.7.0**&#x200B;以降で利用できます。

新しいビデオビューアを有効または無効にするには、Adobe カスタマーケアにお問い合わせください。

## 新しいビデオビューアのプレビュー {#preview}

アセットの詳細ページから新しいビデオビューアをプレビューするには、次の手順を実行します。

1. **Assets** > **ファイル**&#x200B;に移動し、ビデオアセットを含むフォルダーを開きます。
2. ビデオアセットをクリックして、アセットの詳細ページを開きます。
3. 左側のパネルで、**ビューア**&#x200B;をクリックします。
4. **ビューアー** パネルで、**ビデオ （新規）**&#x200B;を選択します。
5. 「**URL**」をクリックして、プレビューリンクをコピーします。
   ![URLをコピー](assets/Copy-url1.jpg)

## Sitesでの新しいビデオビューアの使用 {#use-in-sites}

新しいビデオビューアは、AEM Sitesの既存の&#x200B;**Dynamic Media** コンポーネントを通じて利用できます。

### Dynamic Media コンポーネントの追加

Dynamic Media コンポーネントを使用してビデオを追加するには、次の手順を実行します。

1. **サイトエディター**&#x200B;でページを開きます。
2. **Dynamic Media** コンポーネントをページ上の必要な場所にドラッグします。
3. ページ上の&#x200B;**Dynamic Media** コンポーネントを選択します。
4. コンポーネントをクリックして、アセットセレクターを開きます。
5. ビデオアセットを選択します。

![Dynamic Media コンポーネントをドラッグ &#x200B;](assets/drag-component.jpeg)

### ビューアの設定

ビューアプリセットを設定するには、次の手順を実行します。

1. ページ上の&#x200B;**Dynamic Media** コンポーネントを選択します。
2. コンポーネントツールバーの「**Configure**」をクリックします。
   ![Dynamic Media設定を開く](assets/configure-asset.png)

3. **Dynamic Media設定ダイアログ**&#x200B;で、**ビューアプリセット** ドロップダウンリストから&#x200B;**ビデオ（新規）**&#x200B;を選択します。
   ![&#x200B; ビデオ（新規）ビューアプリセットを選択](assets/viewer-preset.jpeg)

4. 必要な修飾子を&#x200B;**ビューア修飾子** フィールドに入力します（例：`autoplay=true&muted=true`）。
   ![&#x200B; ビューア修飾子](assets/additional-modifiers.jpeg)

5. 変更を保存します。

新しいビデオビューアを使用して、ページにビデオが読み込まれます。

> **注：**&#x200B;新しいビデオ ビューアは、既存のビデオを自動的に置き換えることはできません。 ユーザーは、Dynamic Media コンポーネントを使用する場合、**ビューアプリセット**&#x200B;で&#x200B;**ビデオ（新規）**&#x200B;を手動で選択するか、必要に応じて新しいビデオビューアを指すようにダイレクト URLを更新する必要があります。

### ダイレクト URLを使用したビデオの移行

Dynamic Media コンポーネントではなく直接URLを介してビデオにアクセスする場合は、URLを更新して新しいビデオビューアに切り替えることができます。 例：`https://s7d1.scene7.com/dmviewers/html5/VideoViewer.html?asset=<video-asset>`

## ビューア修飾子 {#viewer-modifiers}

ビューア修飾子を使用すると、アセットの読み込み、再生動作、ストリーミング形式の選択、ビューアの表示を制御できます。

| 修飾子 | 説明 |
|--------|-------------|
| `asset` | ビデオまたはアダプティブビデオセットのアセット IDを指定します。 |
| `posterimage` | 再生開始前に表示される画像を指定します。 |
| `serverurl` | 画像サービングルートパスを指定します。 |
| `contenturl` | コンテンツのルートパスを指定します。 |
| `videoserverurl` | ビデオサーバーのルートパスを指定します。 |
| `sources.dash` | 再生用のDASH マニフェスト URLを指定します。 |
| `sources.hls` | 再生用のHLS マニフェスト URLを指定します。 |
| `autoplay=true` | ビデオが読み込まれると、自動的に再生を開始します。 |
| `controls=true/false` | ビデオ再生コントロールの表示/非表示を切り替えます。 |
| `loop=true` | ビデオの終了後、再生を自動的に再開します。 |
| `muted=true` | ミュート状態で再生を開始します。 |
| `playbackrates` | 使用可能な再生速度オプションを指定します。 |
| `playback` | ストリーミング形式（自動、hls、ダッシュ、またはプログレッシブ）を指定します。 |
| `progressivebitrate` | プログレッシブ再生のビットレートを指定します。 |
| `initialbitrate` | アダプティブストリーミングの初期ビットレートを指定します。 |
| `isletterboxed=true/false` | ビデオをレターボックス化するか、引き伸ばすかを制御します。 |
| `customcss` | ビューアのスタイル設定のためのカスタム CSS ファイルを指定します。 |
| `transition` | ビューア コントロールの表示/非表示の切り替え動作を指定します。 |
| `mode=smartcrop` | ビデオのスマート切り抜き出力を表示します。 |

修飾子は、**ビューア修飾子** フィールドでクエリパラメーターとして指定されます。

## サポートされているイベント {#supported-events}

新しいビデオビューアは、再生中に次のイベントを発生させます。

| イベントタイプ | 説明 |
|-----------|-------------|
| プレイ | 動画の再生を開始します |
| 一時停止 | ビデオが一時停止されています |
| シーク | ビデオ内でのユーザーのシーク |
| 読み込み | 動画が読み込まれました |
| 閉じる | Playerが閉じています |
| メタデータ | 期間などのメタデータ |
| マイルストン | 再生のマイルストーンに達しました |
| current_time | 定期再生位置 |
| フルスクリーン | 全画面表示モードを入力 |
| un_fullscreen | 全画面表示を終了 |

## 親ウィンドウでのイベントの処理 {#handling-events}

新しいビデオビューアは、ビデオインタラクション中に再生関連のメッセージを親ページに送信します。

これらのイベントを処理するには、親アプリケーションがデータを処理する前に、ブラウザーのメッセージイベントをリッスンし、メッセージの発信元を検証する必要があります。

イベントペイロードには、イベントタイプ、再生状態、現在の再生時間、追加メタデータなどの情報が含まれます。 これらのイベントは、分析の追跡、カスタムインタラクション、外部システムとの統合をサポートするために使用できます。

Adobeでは、信頼できるDynamic Media ドメインからのみイベントが処理されるように、メッセージの送信元を検証することをお勧めします。

## 新しいビデオビューアのビデオエンゲージメントレポート {#video-engagement-report}

ビデオエンゲージメントレポートは、Dynamic Mediaの新しいビデオビューアを使用して再生されたビデオの分析指標を提供します。 このレポートは、指定された月の集計パフォーマンスデータを提供し、月次レポートをサポートします。

レポートはリクエストに応じて生成されます。 レポートをリクエストするには、[&#x200B; サポートチケット &#x200B;](https://helpx.adobe.com/jp/enterprise/using/support-for-experience-cloud.html)を作成し、次の詳細を入力します。

* レポートの月 – レポートが必要な月（例：2026年1月）を指定します。
* 配信メールアドレス – レポートを配信するグループ（推奨）または個人のメールアドレス

このレポートでは、視聴数、インプレッション、視聴時間、完了率、エンゲージメントスコアなど、動画ごとのエンゲージメント指標が提供されます。

### レポート形式

* レポートはCSV形式で配信されます。
* 各行は1つのビデオを表します。
* 選択したレポート期間の指標が集計されます。
* 削除されたアセットはレポートから除外されます。
* `tenant_name`によるフィルタリングをサポートしています。

### レポートフィールド

ビデオエンゲージメントレポートには、次のフィールドが含まれます。

| フィールド | 説明 | 計算 |
|-------|------------|-------------|
| `video_id` | 一意のビデオ ID: | 該当なし |
| `video_name` | ビデオアセットの名前 | 該当なし |
| `video_created_date` | ビデオの作成日。 | 該当なし |
| `duration_in_seconds` | ビデオのデュレーション（秒単位）。 | 該当なし |
| `video_views` | 選択したレポート期間中のビデオ再生イベントの合計数。 | 該当なし |
| `video_impressions` | ビデオが読み込まれた合計回数。 | 該当なし |
| `video_watched_seconds` | すべてのプレイイベントで視聴された合計秒数。 | すべてのプレイイベント全体で監視される秒数 |
| `play_rate` | ビデオの読み込みに対するビデオ再生の割合。 | （`video_views` ÷ `video_impressions`） × 100 |
| `avg_time_watched_in_seconds` | 1 ビューあたりの平均視聴時間（秒）。 | `video_watched_seconds` ÷ `video_views` |
| `avg_completion_rate` | ビデオが完全に完了した回数の割合。 | （完了ビュー数÷ `video_views`） × 100 |
| `engagement_score` | すべてのプレイイベントの平均視聴率。 | （すべてのセッションで視聴されたビデオタイムラインの合計割合÷ `video_views`） |
| `tenant_name` | データに関連付けられている会社またはテナントの識別子。 | 該当なし |

## よくある質問 {#faq-video-engagement}

+++ビデオが自動再生に設定されている場合、自動的にビューとしてカウントされますか、またはユーザーが最小限のデュレーションを視聴した後にのみカウントされますか？

自動再生はビデオビューとしてカウントされます。 自動的に開始された再生は、ビューとして記録されます。

+++

+++ユーザーがビデオの一部のみを視聴した場合（例えば、10秒のビデオの最初の2秒と最後の2秒）、完了したビューとしてカウントされますか？

ビデオの一部がスキップされた場合でも、再生がビデオタイムラインの最後に達すると、ビューは完了としてカウントされます。

+++

+++ユーザーが後ろにスクラブしてビデオの一部を再視聴した場合、video_views数、engagement_score、またはその両方が増加しますか？

ビデオの一部を再視聴しても、video_views数は増加しません。 追加の再生はengagement_scoreに貢献します。

+++

+++同じユーザーがページを再読み込みせずに同じビデオを複数回視聴した場合、video_viewsとengagement_scoreはどのように計算されますか？

ページを再読み込みせずに再生を繰り返しても、video_views数は増えません。 追加の再生はengagement_scoreに貢献します。

+++

+++動画の一時停止や再開は、エンゲージメントの追跡や完了率の計算に影響しますか？

再生を一時停止して再開しても、エンゲージメントの追跡や完了率の計算には影響しません。

+++
