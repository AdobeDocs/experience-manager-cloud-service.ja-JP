# AEM Assets アクティベーションレポート（Beta）：指標の定義

## アセットパフォーマンス指標の理解

AEM Assetsシステム内のあらゆるアセットは、アップロードから承認、商品画像、共有リンク、ヒーローバナーの作成に至るまで、ストーリーを語ります。 そのジャーニーは何十億というブランドのインプレッションを促し、それを理解することは、アセットへの投資価値を引き出す鍵となります。

この記事では、チームがアセットを使用して実際に作業する方法をグループ化し、そのジャーニーの背後にある指標を詳しく説明します。

- **アクティベーション（従業員およびパートナーが開始）**:AEM Assets（Assets ビューおよびContent Hub）全体のダウンロード、共有、アクティベーション。
- **ディストリビューション（外部からエンドユーザー）**:Dynamic MediaやAEM Sitesなどのチャネルを通じて世界中に配信します。

これらの指標を組み合わせることで、社内のアセット活動を外部のリーチやエンゲージメントに結びつけ、コンテンツ運用がどのように実際のビジネス成果に結びついているのかを確認することができます。

これは、Adobe AEM Assetsを活用する出発点にすぎません。これらの指標は、今後も進化し続けることでしょう。 今後もこのレポートをより有用なものにできるように、ぜひフィードバックをお寄せください。

**メモ**:Adobe Product/Engineeringでこれらを確認したい場合、またはフィードバックがある場合は、`GRP-AEM-DAM-METRICS-BETA@ADOBE.COM`にメールを送信します。

| | 指標 | 定義 |
|---|---|---|
| **アクティベーション** | **ダウンロード （Assets ビュー）** | 例えば、社内ユーザーがAssets ビューのインターフェイスを通じてファイルを参照およびダウンロードするなど、AEM Assetsのコア環境からアセットを直接ダウンロードした回数。 |
| | **共有（Assets ビュー）** | 外部または未認証のユーザーにログインすることなく、特定のアセット、フォルダー、コレクションへの直接アクセスを提供するために、AEM Assets ビュー内から（生成された共有可能なリンクを介して）共有の数。 |
| | **ダウンロード （Content Hub）** | Content Hubを通じて、より広範な部門が承認済みのブランドに即したアセットを参照およびダウンロードできるセルフサービス UIを通じて、ユーザーがアセットをダウンロードする回数。 |
| | **シェア （Content Hub）** | 他のユーザーが選択したアセットまたはコレクションに直接アクセスできるように、Content Hub内から（生成された共有可能なリンクを介して）共有の数。 |
| | **Content Hubにアクティベート** | AEM Assetsからアセットを承認/公開して、Content Hubのエンドユーザーが利用できるようにするアクション。 承認済みアセットのみがContent Hubに表示されます。 |
| | **Dynamic Mediaへのアクティベート** | AEM AssetsからDynamic Media サービスにアセットを公開し、レンディションに処理して、Dynamic Mediaの画像/動画パイプラインおよびCDNを介してリアルタイムで配信できるようにするアクション。 |
| **配布** | **Dynamic Media配信リクエスト** | Dynamic Mediaを介して、すべてのチャネルでアセットがリクエストおよび配信された合計回数。 |
| | **Dynamic Media Unique Assetsが提供されました** | 指定された期間にDynamic Mediaを介して少なくとも1回は配信された個別のアセットの数。リクエストされた回数や配信されたレンディションの数に関係なく、アセットごとに1回カウントされます。 |
| | **Dynamic Mediaの一意の変換** | 特定の期間にDynamic Mediaを介して配信された、アセットの異なるバリエーションの数。 これは、デバイス、画面サイズ、ユースケースをまたいで、コンテンツがどのように適応しているかを示しています。 |
| | **サイトに公開** | AEM Assetsからアセットを公開して、ライブのAEM Sites ページで使用できるようにすること。 |
