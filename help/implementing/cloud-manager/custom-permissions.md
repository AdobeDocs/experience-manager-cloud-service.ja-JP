# カスタム権限 {#custom-permissions}

カスタム権限を使用して、設定可能な権限を持つ新しいカスタム権限プロファイルを作成し、Cloud Manager ユーザーのプログラム、パイプライン、環境へのアクセスを制限する方法について説明します。

>[!NOTE]
>
>この機能は、次の場合にのみ使用できます。 [アーリーアダプタープログラム。](/help/implementing/cloud-manager/release-notes/current.md#early-adoption)

## はじめに {#introduction}

Cloud Manager には、Cloud Manager の様々な機能へのアクセスを制御する、事前定義済みのロールのセットがあります。

* ビジネスオーナー
* プログラムマネージャー
* デプロイメントマネージャー
* デベロッパー

カスタム権限を使用すると、Cloud Manager ユーザーがプログラム、パイプライン、環境にアクセスできるように、権限を設定して新しいカスタム権限プロファイルを作成できます。

>[!TIP]
>
>事前定義済みのロールについて詳しくは、ドキュメントを参照してください。 [AEMas a Cloud Serviceのチームおよび製品プロファイル。](/help/onboarding/aem-cs-team-product-profiles.md)

## カスタム権限の使用 {#using}

独自のカスタム権限を作成して使用するには、次の 3 つの手順が必要です。

1. [新しい製品プロファイルを作成します。](#create)
1. [新しい製品プロファイルにカスタム権限を割り当てます。](#assign-permissions)
1. [新しい製品プロファイルにユーザーを割り当てます。](#assign-users)

この節では、これらの手順について詳しく説明します。 これは、 [キーワード](#terms) および [設定可能な権限](#configurable-permissions) セクションを開き、独自のカスタム権限を作成します。

>[!NOTE]
>
>新しいプロファイルを作成し、Cloud Manager の権限を管理するには、Adobe Experience Manager as a Cloud ServiceのAdmin Consoleの製品管理者権限が必要です。

### 新しい製品プロファイルの作成 {#create}

カスタム権限を割り当てる前に、製品プロファイルを作成する必要があります。

1. Cloud Manager( ) にログインします。 [my.cloudmanager.adobe.com](https://my.cloudmanager.adobe.com/)

1. Cloud Manager のランディングページで、 **アクセスを管理** ボタン

![「アクセスを管理」ボタン](assets/manage-access.png)

1. 次のページにリダイレクトされます： **製品** Admin Consoleのタブを使用して、cloud manager のユーザーと権限を管理できます。 Admin Consoleで、 **新しいプロファイル** 」ボタンをクリックします。

![新規プロファイルボタン](assets/admin-console-new-profile.png)

1. プロファイルに関する一般的な詳細を入力します。

   * **製品プロファイル名**  — プロファイルのわかりやすい名前
   * **表示名** - UI（オプション）に表示される省略名
   * **説明**  — 目的を説明するプロファイルのわかりやすい説明（オプション）
   * **電子メールでユーザーに通知**  — 選択すると、ユーザーは、このプロファイルに追加または削除されたときに電子メールで通知されます。

1. 選択 **保存** 完了したら、

新しい製品プロファイルが保存され、Admin Consoleの製品プロファイルのリストに表示されます。

### プロファイルへのカスタム権限の割り当て {#assign-permissions}

これで、新しい製品プロファイルが作成されたので、そのプロファイルにカスタム権限を割り当てることができます。

1. Admin Consoleで、 [先ほど作成した新しい製品プロファイル。](#create)

1. 表示されるウィンドウで、「 **権限** タブをクリックして、編集可能な権限のリストを表示します。

   ![編集可能な権限](assets/permissions-tab.png)

1. を選択します。 **編集** 編集する権限のリンク。

1. The **権限を編集** ウィンドウが開きます。
   * 前の手順で選択した権限が左の列で選択されます。
   * 権限の割り当てに使用できる権限項目は、中央の列に表示されます。 **使用可能な権限** 項目。
   * 割り当てられた権限項目は、右側の列に「 **含まれる権限項目**.

   ![権限項目を編集](assets/edit-permission-items.png)

1. プラス (`+`) アイコンをクリックして、列に追加します。 **含まれる権限項目**.

   * を選択します。 `i` 権限項目の横にあるアイコンをクリックして、詳細情報を確認します。

1. を選択します。 **すべて追加** ボタン **使用可能な権限** 列を使用して、すべての権限を追加します。

1. 選択 **保存** 新しい製品プロファイルの権限項目の定義が完了したら、次の手順に従います。

これで、新しい製品プロファイルがカスタム権限で保存されました。

### カスタム権限へのユーザーの割り当て {#assign-users}

これで、カスタム権限で作成した新しい製品プロファイルにユーザーを割り当てることができるようになりました。

1. Admin Consoleで、 [カスタム権限を割り当てた新しい製品プロファイル。](#assign-permissions)

1. 表示されるウィンドウで、「 **ユーザー** タブをクリックします。

1. を選択します。 **ユーザーを追加** ボタンをクリックし、カスタム権限を持つユーザーを新しい製品プロファイルに割り当てます。

の節を参照してください。 **製品プロファイルへのユーザーおよびユーザーグループの追加** ドキュメントの [エンタープライズユーザー向け製品プロファイルの管理](https://helpx.adobe.com/jp/enterprise/using/manage-product-profiles.html) を参照してください。

## 設定可能な権限 {#configurable-permissions}

カスタムプロファイルの作成に使用できる権限は次のとおりです。

| 権限 | 説明 |
|---|---|
| プログラムの作成 | ユーザーにプログラムの作成を許可 |
| プログラムアクセス | ユーザーがプログラムにアクセスすることを許可 |
| プログラムの編集 | ユーザーにプログラムの編集を許可 |
| 環境の作成 | ユーザーに環境の作成を許可 |
| 環境の編集 | ユーザーが環境を更新および編集できるようにする |
| 環境ログ読み取り | ユーザーに環境ログの読み取りを許可 |
| 環境変数管理 | 環境設定の作成、編集、削除をユーザーに許可 |
| 環境復元作成 | ユーザーに環境の復元の作成を許可 |
| 開発環境の迅速なリセット | ユーザーが迅速な開発環境をリセットすることを許可 |
| コンテンツコピー管理 | ユーザーがコンテンツのコピー操作を管理することを許可する |
| パイプラインの作成 | ユーザーに新しいパイプラインの作成を許可 |
| パイプラインの削除 | ユーザーにパイプラインの削除を許可 |
| パイプライン編集 | ユーザーによるパイプラインの編集を許可 |
| 実稼動デプロイメントの承認/却下 | ユーザーが実稼動デプロイメントステップを承認または拒否することを許可 |
| パイプライン実行のキャンセル | ユーザーがパイプライン実行をキャンセルすることを許可 |
| パイプライン実行開始 | ユーザーが新しいパイプライン実行を開始することを許可 |
| 重要な指標の失敗を上書き/拒否 | 重要な指標のエラーの上書き/拒否をユーザーに許可 |
| 実稼動デプロイメントスケジュール | ユーザーが実稼動のデプロイメントステップをスケジュールすることを許可 |
| リポジトリ情報アクセス | ユーザーがリポジトリ情報にアクセスし、アクセスパスワードを生成することを許可 |
| リポジトリ作成 | ユーザーに新しい Git リポジトリーの作成を許可 |
| リポジトリの削除 | Git リポジトリの削除をユーザーに許可 |
| リポジトリの編集 | ユーザーに Git リポジトリの編集を許可 |
| リポジトリコード生成 | ユーザーがアーキタイプからプロジェクトを生成することを許可 |
| ドメイン名管理 | ユーザーがドメイン名を作成、編集、削除することを許可 |
| IP 管許可リストに加える理 | IP バインディングおよび IP バインディングの作成/編集/許可リストに加える削除をユーザーに許可リストに加える許可 |
| ネットワークインフラストラクチャの管理 | ネットワークインフラストラクチャの作成/編集/削除をユーザーに許可 |
| SSL 証明書管理 | ユーザーに対し、SSL 証明書の作成、編集、削除を許可 |
| New Relic Sub Account User Manager | ユーザーに対し、新しい relic サブアカウントユーザーの読み取り/編集を許可 |

### 組織レベルの権限 {#organization-level}

組織レベルの権限とは、組織内のすべてのプログラムに対して常に付与される権限を指します。

次の権限は、組織レベルの権限です。

* **プログラムの作成**  — この権限を持つユーザーは、組織内にプログラムを作成できます。
* **リポジトリ情報アクセス** このテナント/組織レベルの権限を使用すると、ユーザーは、アクセスして顧客プロジェクトに投稿するためのユーザー名、パスワード、リポジトリ URL を生成できます。
   * リポジトリアクセスのユーザー名とパスワードは、組織内のすべてのリポジトリで共通ですが、リポジトリの URL は各プログラムに固有です。
   * 詳しくは、 [リポジトリへのアクセス](/help/implementing/cloud-manager/managing-code/accessing-repos.md) を参照してください。

## キーワード {#terms}

カスタム権限と事前定義された役割の作成および管理では、次の用語が使用されます。

| 用語 | 説明 |
|---|---|
| 定義済みの権限 | 次のような事前定義済みの役割 **ビジネスオーナー**, **デプロイメントマネージャー**&#x200B;などを使用して、Cloud Manager の様々な機能を管理します。 事前定義済みのロールについて詳しくは、ドキュメントを参照してください。 [AEMas a Cloud Serviceのチームおよび製品プロファイル。](/help/onboarding/aem-cs-team-product-profiles.md) |
| カスタム権限 | Cloud Manager の機能。ユーザーは権限プロファイルを作成して、Cloud Manager のサポートされる機能を管理する役割を定義できます。 |
| 製品プロファイル | Admin Console で作成され、権限プロファイルの一部であるユーザーに適用できる設定可能な権限を管理します |
| 設定可能な権限 | 権限プロファイルで設定できる Cloud Manager 権限 |
| 権限項目 | 権限を適用できるプログラム、環境またはパイプラインリソース |

権限項目は、権限が適用される範囲を指します。 通常は、次のいずれかになります。

| 権限項目タイプ | 例 | 説明 |
|---|---|---|
| 組織 | organization:companyA | 組織の該当するすべてのリソース。 リソースには、プログラム、環境、パイプラインなどがあります。 ユーザーが任意の権限の組織を追加した場合、その組織内の新しいリソースもすべてその権限を持ちます。 |
| プログラム | プログラム A | プログラムのすべての適用可能なリソース |
| 環境 | プログラム A ：環境 | 特定の環境に適用可能 |
| パイプライン | プログラム A ：パイプライン | 特定のパイプラインに適用可能 |

## 制限事項 {#limitations}

カスタム権限を使用する場合は、次の制限事項に注意してください。

* また、カスタム権限プロファイルは、権限を設定する際に AMS プログラム、環境、パイプラインをリストします。
* Cloud Manager で作成したプログラム、環境、パイプラインなどのリソースが権限設定のためにAdmin Consoleに表示されるまでに最大 2 分かかる場合があります。
* カスタム権限サービスが応答できない場合でも、事前定義済みのプロファイルが引き続き使用でき、事前定義済みのプロファイルのユーザーが適切なアクセス権を持つことがあります。

## よくある質問 {#faq}

### 事前定義された権限プロファイルはどれですか？

* ビジネスオーナー
* プログラムマネージャー
* デプロイメントマネージャー
* デベロッパー

事前定義済みの役割について詳しくは、 [AEMas a Cloud Serviceのチームおよび製品プロファイル。](/help/onboarding/aem-cs-team-product-profiles.md)

### カスタムプロファイルの概要を含む、事前定義済みの権限プロファイルへの影響

デフォルトの製品プロファイルと Cloud Manager の役割は、引き続き以前と同じように機能します。

### 事前定義済みの権限プロファイルを編集できますか？

いいえ、デフォルトのプロファイルは編集できません。 デフォルトの権限プロファイルに対する権限を追加または削除することはできません。 ユーザーは、事前定義済みのプロファイルに対してのみ追加または削除できます。

### カスタムプロファイルが利用可能になったので、事前定義済みの権限プロファイルを削除する必要はありますか？

定義済みの権限プロファイルは、Admin Consoleから削除できません。

### ユーザーを複数の権限プロファイルに追加できますか？

はい。ユーザーは、事前定義済みおよびカスタム権限プロファイルを含む、複数のプロファイルに属することができます。 ユーザーが複数のプロファイルに割り当てられると、割り当てられたすべての権限プロファイルの組み合わせ権限がそのユーザーに対して使用できるようになります。

### ユーザーが環境/パイプラインを編集する権限を持っているが、環境/パイプラインを含むプログラムへのアクセス権を持っていない場合、どうなりますか？

この場合、ユーザーは、 **プログラムアクセス** 環境またはパイプラインを含む権限。
