# カスタム権限 {#custom-permissions}

Cloud Managerでカスタマイズされた権限プロファイルを設定する方法について説明します。 プログラム、パイプライン、環境に対して特定のアクセス制御を設定し、各ユーザーが実行できる処理を詳細に制御できます。

## はじめに {#introduction}

Cloud Manager には、Cloud Manager の様々な機能へのアクセスを制御する、事前定義済みの役割のセットがあります。

* ビジネスオーナー
* プログラムマネージャー
* デプロイメントマネージャー
* 開発者

カスタム権限を使用すると、Cloud Manager ユーザーのアクセスをプログラム、パイプライン、環境に制限するための設定可能な権限を持つカスタム権限プロファイルを作成できます。

>[!TIP]
>事前に定義された役割について詳しくは、[AEMas a Cloud Service のチームおよび製品プロファイル](/help/onboarding/aem-cs-team-product-profiles.md)を参照してください。

## カスタム権限の使用 {#using}

独自のカスタム権限を作成して使用するには、次の 3 つの手順が必要です。

1. [製品プロファイルを作成します](#create)。
1. [製品プロファイルにカスタム権限を割り当てます](#assign-permissions)。
1. [製品プロファイルにユーザーを割り当てます](#assign-users)。

>[!TIP]
>独自のカスタム権限を作成するには、[条件](#terms)および[設定可能な権限](#configurable-permissions)のセクションを確認してください。

>[!IMPORTANT]
>Cloud Managerの製品プロファイルを作成し、権限を管理するには、Admin Console Adobe Experience Manager as a Cloud Service版で製品管理者権限を持っている必要があります。

### 製品プロファイルの作成 {#create}

{{sign-in-to-cloud-manager}}

1. Cloud Manager ランディングページで、**アクセスの管理**&#x200B;をクリックします。

   ![「アクセスを管理」ボタン](assets/manage-access.png)

1. Admin Consoleで、**新規プロファイル**&#x200B;をクリックします。

   ![「新規プロファイル」ボタン](assets/admin-console-new-profile.png)

1. プロファイルに関する一般的な詳細を入力します。

   * **製品プロファイル名** - プロファイルのわかりやすい名前
   * **表示名** - UI に表示される省略名（オプション）
   * **説明** - 目的を説明するプロファイルのわかりやすい説明（オプション）
   * **メールでユーザーに通知** - このプロファイルに対して追加または削除が行われたユーザーへのメール通知。

1. 「**保存**」をクリックします。

新しい製品プロファイルが保存され、Admin Console の製品プロファイルのリストに表示されます。

### 製品プロファイルへのカスタム権限の割り当て {#assign-permissions}

1. Admin Consoleで、作成した[新しい製品プロファイルの名前](#create)をクリックします。

1. **カスタムプロファイル** ページで、「**権限**」タブをクリックします。

   ![編集可能な権限](assets/permissions-tab.png)

1. 権限名の行で、**編集**&#x200B;をクリックします。

1. カスタムプロファイル **の権限を編集ダイアログボックスで、次のいずれかの操作を行います。**

   * **使用可能な権限項目**&#x200B;列の上部付近で、![追加アイコンまたはプラス記号アイコン &#x200B;](https://spectrum.adobe.com/static/icons/workflow_18/Smock_AddCircle_18_N.svg) **すべてを追加**&#x200B;をクリックしてすべての権限を含めます。
   * **含まれる権限項目**&#x200B;列に1つの権限を追加するには、関連付けられている![追加アイコンまたはプラス記号アイコン &#x200B;](https://spectrum.adobe.com/static/icons/workflow_18/Smock_AddCircle_18_N.svg)をクリックします。

     ![権限項目の編集](assets/edit-permission-items.png)

1. 「**保存**」をクリックします。

### 製品プロファイルへのユーザーの割り当て {#assign-users}

1. Admin Consoleで、カスタム権限を割り当てた[新製品プロファイルの名前](#assign-permissions)をクリックします。

1. 開いたウィンドウで、「**ユーザー**」タブをクリックします。

1. 「**ユーザーを追加**」をクリックし、カスタム権限を持つ製品プロファイルにユーザーを割り当てます。

Admin Consoleの使用方法について詳しくは、[&#x200B; エンタープライズユーザー向け製品プロファイルの管理](https://helpx.adobe.com/jp/enterprise/using/manage-product-profiles.html)で、**製品プロファイルへのユーザーとユーザーグループの追加**&#x200B;を参照してください。

## 設定可能な権限 {#configurable-permissions}

カスタム製品プロファイルを作成する際は、次の権限を使用できます。

| 権限 | 説明 |
| --- | --- |
| `Program Create` | ユーザーがプログラムを作成できるようにします。 |
| `Program Access` | ユーザーがプログラムにアクセスできるようにします。 |
| `Program Edit` | ユーザーがプログラムを編集できるようにします。 |
| `Environment Create` | ユーザーが環境を作成できるようにします。 |
| `Environment Edit` | ユーザーが環境を更新および編集できるようにします。 |
| `Environment Logs Read` | ユーザーが環境ログを読み取れるようにします。 |
| `Environment Variables Manage` | ユーザーが環境設定を作成／編集／削除できるようにします。 |
| `Environment Restore Create` | ユーザーが環境の復元を作成できるようにします。 |
| `Rapid Development Environment Reset` | ユーザーが高速開発環境（RDE）をリセットできるようにします。 |
| `Content Copy Manage` | ユーザーがコンテンツのコピー操作を管理できるようにします。 |
| `Pipeline Create` | ユーザーがパイプラインを作成できるようにします。 |
| `Pipeline Delete` | ユーザーがパイプラインを削除できるようにします。 |
| `Pipeline Edit` | ユーザーがパイプラインを編集できるようにします。 |
| `Production Deployments Approve/Reject` | ユーザーが実稼動デプロイメント手順を承認または却下できるようにします。 |
| `Pipeline Executions Cancel` | ユーザーがパイプライン実行をキャンセルできるようにします。 |
| `Pipeline Executions Start` | ユーザーが新しい実行パイプラインを開始できるようにします。 |
| `Override/Reject Important Metric Failures` | ユーザーが重要な指標エラーを上書き／却下できるようにします。 |
| `Production Deployments Schedule` | ユーザーが実稼動デプロイメント手順をスケジュールできるようにします。 |
| `Repository Info Access` | ユーザーがリポジトリ情報にアクセスし、アクセスパスワードを生成できるようにします。 |
| `Repository Create` | ユーザーが Git リポジトリを作成できるようにします。 |
| `Repository Delete` | ユーザーが Git リポジトリを削除できるようにします。 |
| `Repository Edit` | ユーザーが Git リポジトリを編集できるようにします。 |
| `Repository Code Generate` | ユーザーがアーキタイプからプロジェクトを生成できるようにします。 |
| `Domain Name Manage` | ユーザーがドメイン名を作成／編集／削除できるようにします。 |
| `IP Allowlist Manage` | ユーザーが IP 許可リストと IP 許可リストバインディングを作成／編集／削除できるようにします。 |
| `Network Infrastructure Manage` | ユーザーがネットワークインフラストラクチャを作成、削除、編集、テストできるようにします。 |
| `SSL Certificate Manage` | ユーザーが SSL 証明書を作成／編集／削除できるようにします。 |
| `New Relic Sub Account User Manage` | ユーザーが New Relic サブアカウントユーザーの読み取り／編集をできるようにします。 |

### 組織レベルの権限 {#organization-level}

組織レベルの権限とは、組織内のすべてのプログラムで常に与えられる権限を指します。

次の権限は、組織レベルの権限です。

* **`Program Create`** – この権限を持つユーザーは、組織内でプログラムを作成できます。
* **リポジトリ情報アクセス** – このテナント/組織レベルの権限を持つユーザーは、顧客プロジェクトにアクセスして貢献するためのユーザー名、パスワード、リポジトリ URLを生成できます。
   * リポジトリアクセス用のユーザー名とパスワードは、組織内のすべてのリポジトリで共通です。 ただし、リポジトリ URL はプログラムごとに異なります。
   * 詳しくは、[リポジトリへのアクセス](/help/implementing/cloud-manager/managing-code/accessing-repos.md)を参照してください。

## 用語 {#terms}

次の用語は、カスタム権限と事前定義済みの役割の作成と管理に使用されます。

| 用語 | 説明 |
| --- | --- |
| 定義済みの権限 | Cloud Manager の様々な機能を管理するための、**ビジネスオーナー**&#x200B;や&#x200B;**デプロイメントマネージャー**&#x200B;といった事前定義済みの役割。 事前に定義された役割について詳しくは、[AEM as a Cloud Service のチームおよび製品プロファイル](/help/onboarding/aem-cs-team-product-profiles.md)を参照してください。 |
| カスタム権限 | Cloud Manager の機能により、ユーザーは権限プロファイルを作成して、Cloud Manager でサポートされる機能を管理する役割を定義できます。 |
| 製品プロファイル | Admin Console で作成され、権限プロファイルに含まれるユーザーに適用される設定可能な権限を管理します。 |
| 設定可能な権限 | 権限プロファイルで設定できる Cloud Manager の権限。 |
| 権限項目 | 権限を適用できるプログラム、環境またはパイプラインリソース。 |

権限項目とは、権限が適用される範囲を指します。 通常は次のいずれかです。

| 権限項目タイプ | 例 | 説明 |
| --- | --- | --- |
| 組織 | 組織:companyA | 組織の適用可能なすべてのリソース。 リソースには、プログラム、環境、パイプラインなどがあります。 ユーザーが任意の権限を得るために組織を追加すると、その組織内のすべての新しいリソースにもその権限が付与されます。 |
| プログラム | プログラム A | プログラムの適用可能なすべてのリソース。 |
| 環境 | プログラム A：環境 | 特定の環境に適用可能です。 |
| パイプライン | プログラム A：パイプライン | 特定のパイプラインに適用可能です。 |

## 使用上のメモ {#usage-notes}

* また、カスタム権限プロファイルは、権限を設定する際に AMS プログラム、環境、パイプラインをリストします。
* Cloud Managerで作成されたプログラム、環境、パイプラインなどのリソースをAdmin Consoleで表示するには、数分かかります。
* カスタム権限サービスが応答しない場合、事前定義済みのプロファイルは引き続き使用でき、事前定義済みプロファイルのユーザーには適切なアクセス権が付与されます。

## よくある質問 {#faq}

### 定義済みの権限プロファイルはどれですか？

* ビジネスオーナー
* プログラムマネージャー
* デプロイメントマネージャー
* 開発者

事前に定義された役割について詳しくは、[AEM as a Cloud Service のチームおよび製品プロファイル](/help/onboarding/aem-cs-team-product-profiles.md)を参照してください。

### カスタムプロファイルを導入すると、定義済みの権限プロファイルはどうなりますか？

デフォルトの製品プロファイルとCloud Managerの役割は、引き続き以前と同様に機能します。

### 事前定義済みの権限プロファイルを編集できますか？

いいえ。 デフォルトのプロファイルは編集できません。 デフォルトの権限プロファイルに権限を追加または削除することはできません。 ユーザーを追加または削除できるのは、事前定義済みのプロファイルのみです。

### カスタムプロファイルが使用可能になったので、事前定義済みの権限プロファイルを削除する必要はありますか？

Admin Consoleから事前定義済みの権限プロファイルを削除しないでください。

### ユーザーを複数の権限プロファイルに追加できますか？

はい。 ユーザーは、定義済みの権限プロファイルやカスタム権限プロファイルを含む複数のプロファイルに割り当てることができます。 ユーザーが複数のプロファイルに割り当てられている場合、そのユーザーは、割り当てられているすべての権限プロファイルの権限を組み合わせて使用できるようになります。

### ユーザーが環境/パイプラインを編集する権限を持っているが、環境/パイプラインを含むプログラムにアクセスできない場合はどうなりますか？

環境またはパイプラインに関連付けられた&#x200B;**プログラムアクセス**&#x200B;権限がない場合、ユーザーは環境またはパイプラインにアクセスできません。
