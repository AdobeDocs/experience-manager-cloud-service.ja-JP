# 「ありがとうございます」メッセージとリダイレクト URL の設定

送信後のエクスペリエンスは、ユーザーの満足度とフォームの完了率に大きな影響を与えます。 Adobe ユニバーサルエディターは、パーソナライズされたありがとうメッセージや特定のページへの戦略的なリダイレクトなど、フォームの送信後にユーザーに表示される内容を設定するための包括的なオプションを提供します。

この記事では、フォーム送信の効果を最大限に高めるための技術的な考慮事項、ベストプラクティス、ユーザーエクスペリエンスのガイドラインなど、「ありがとうございます」メッセージとリダイレクト URL の両方を実装する際の詳細なガイダンスを提供します。

## 前提条件

送信後のエクスペリエンスを設定する前に、次のことを確認します。

**技術的セットアップ：**

- 適切な権限を持つユニバーサルエディターへのアクセス
- ユニバーサルエディターで作成された既存のアダプティブフォーム
- 組織のリダイレクト URL の要件を理解する

**計画に関する考慮事項：**

- **メッセージ戦略**：ありがとうメッセージに含めるトーン、長さ、特定の情報を定義します
- **リダイレクト方法**：ターゲットページを特定し、フォーム完了後の完了エクスペリエンスに最適化されるようにします
- **Analytics 統合**：ありがとうメッセージやリダイレクト先とのユーザーインタラクションを追跡する方法を計画します

## 「ありがとうございます」メッセージの設定

「ありがとうございます」メッセージは、フォーム送信の成功を即座に確認するもので、パーソナライズされたコンテンツ、次のステップ、ユーザーの送信に関連する重要な情報などを含めることができます。

### 「ありがとうございます」メッセージを使用するタイミング

「ありがとうございます」メッセージは次の場合に最適です。

- **簡単な確認**：ユーザーは、追加のナビゲーション要件なしに確認が必要です
- **教育コンテンツ**：次の具体的な手順や重要な情報を提供する必要があります
- **ブランドの一貫性**：メッセージは、組織のコミュニケーションスタイルに合わせて作成できます
- **単一ページエクスペリエンス**：ワークフローの継続性のために、ユーザーは現在のページにとどまる必要がある

### 実装手順

**1. フォームのプロパティへのアクセス**

アダプティブフォームをユニバーサルエディターで開き、ツールバーの **フォームプロパティを編集** アイコンをクリックします。 包括的なフォームプロパティダイアログが開きます。

**2. 「ありがとうございます」設定に移動します**

フォームプロパティダイアログで「**ありがとうございました**」タブを選択し、送信後の設定オプションにアクセスします。

**3. メッセージ表示の設定**

使用可能なオプションから **メッセージを表示** を選択します。 これにより、リッチテキスト機能を備えたメッセージコンテンツエディターがアクティブになります。

**4. メッセージコンテンツの作成**

「**メッセージコンテンツ**」フィールドで、リッチテキストエディターを使用して「ありがとうございます」メッセージを作成します。 エディターは、次の項目をサポートしています。

- **テキストの書式設定**：太字、斜体、下線、色のオプション
- **リスト**：情報を整理するための箇条書きと番号付きのリスト
- **リンク**：関連リソースまたは次の手順への直接リンク
- **フルスクリーン編集**：編集ワークスペースを大きくするには、展開アイコンをクリックします

### 技術上の考慮事項

**メッセージの表示動作：**

- メッセージは、フォーム送信直後にモーダルオーバーレイに表示されます
- コンテンツは、HTMLの書式設定をサポートし、レスポンシブデザインを維持します
- メッセージは、ユーザーによって閉じられるか、自動終了タイマーで設定できます

**コンテンツのガイドライン：**

- 必要な情報を提供しながら、メッセージを簡潔に保つ
- 必要に応じて次のステップを明確に含める
- 参照番号または確認の詳細を含めることを検討します
- モバイルに適したフォーマットの確保

### 実装例

     送信いただきありがとうございます。
    
     アプリケーションを受け取り、参照番号#REF-2024-001234 を割り当てました。
    
    **次の手順：**
    - 15 分以内に確認メールが届きます 
    - チームは 2 営業日以内に送信を確認します 
     – 追加情報が必要な場合は直接ご連絡します 
    
    **サポートが必要な場合？**サポートチーム（support@example.com）までお問い合わせください 

## リダイレクト URL の設定

リダイレクト URL を使用すると、フォームの送信後にユーザーが特定のページに自動的に移動し、既存のワークフローとシームレスに統合したり、ユーザーを関連コンテンツにリダイレクトしたりできます。

### リダイレクト URL を使用するタイミング

リダイレクト URL は次の場合に最適です。

- **ワークフロー統合**：ダッシュボード、アカウントページまたはプロセスの次のステップにユーザーを誘導する
- **コンテンツ配信**：フォームの応答に基づいて、関連する製品、サービスまたは情報を表示します
- **Analytics トラッキング**：特定のトラッキング実装を持つページへのリダイレクト
- **複数ステップのプロセス**：複雑なワークフローの次のフェーズへのユーザーの移行

### 実装手順

**1. フォームのプロパティへのアクセス**

アダプティブフォームをユニバーサルエディターで開き、「**フォームのプロパティを編集** アイコンをクリックして、フォーム設定ダイアログを開きます。

**2. 「ありがとうございます」設定に移動します**

リダイレクト設定オプションにアクセスするには、フォームプロパティダイアログの「**ありがとうございました**」タブを選択します。

**3. リダイレクト機能を有効にする**

使用可能な送信後オプションから「**URL にリダイレクト**」を選択します。

**4. 宛先 URL を設定**

入力したフィールドにターゲット URL を入力します。 柔軟な実装のために、システムは複数の URL 形式をサポートしています。

### URL 設定オプション

**絶対 URL**

プロトコルとドメインを含む完全な Web アドレス：

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**相対パス**

現在のドメインに関連するパス：

    /thank-you
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sitesページ参照**

AEM Sites実装内の他のページへの参照：

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 技術上の考慮事項

**リダイレクトの動作：**

- リダイレクトはフォームの送信が成功した直後に行われます
- ブラウザー履歴には、適切なバックボタン機能へのリダイレクトが含まれます
- リダイレクトタイミングはオプションの遅延で設定可能

**URL 検証：**

- システムは、設定を許可する前に URL 形式を検証します
- 相対 URL は現在のドメインに対して解決されます
- 外部 URL は、必要に応じて適切な CORS 設定が必要です

## ベストプラクティスと推奨事項

### ユーザーエクスペリエンスのガイドライン

**メッセージの最適化：**

- **明確さ第一**：ユーザーが自分の送信が成功したことを直ちに理解できるようにします
- **付加価値**：次のステップに役立つ情報を提供します
- **一貫したブランディング**：組織の声とビジュアルスタイルを維持します
- **モバイルでの考慮事項**：様々な画面サイズでメッセージをテストします

**リダイレクトの最適化：**

- **ページの最適化**：フォーム後の訪問者向けにリダイレクトの宛先が最適化されていることを確認する
- **パフォーマンスの読み込み**：リダイレクトページの読み込みをすばやく確認して、ユーザーエクスペリエンスを維持します
- **コンテンツの関連性**：リダイレクトコンテンツがフォームコンテキストに関連していることを確認します

### セキュリティに関する考慮事項

**URL 検証：**

- 悪意のあるリダイレクトを防ぐため、リダイレクト URL に適切な検証を実装する
- 許可されているリダイレクトドメインの許可リストへのアプローチを検討
- 異常なアクティビティに対するリダイレクトパターンの監視

**コンテンツのセキュリティ：**

- 「ありがとうございます」メッセージコンテンツを不要部分に削除して、スクリプトの挿入を防ぎます
- リッチテキストコンテンツに対して適切なコンテンツセキュリティポリシーを実装する
- リダイレクト先の定期的なセキュリティレビュー

### 分析とトラッキング

**実装に関する考慮事項：**

- **目標のトラッキング**:「ありがとうございます」メッセージビューとリダイレクト完了の両方の Analytics 目標を設定します
- **ユーザージャーニーマッピング**：ユーザーによる送信後のエクスペリエンスの操作方法を追跡します
- **コンバージョンの最適化**:A/B テストによって「ありがとうございます」メッセージとリダイレクトの宛先が異なる

**計測戦略：**

- 解雇前に「ありがとうございます」メッセージに費やした時間の監視
- 「ありがとうございます」メッセージ内のリンクのクリックスルー率を追跡
- リダイレクト先ページのユーザー行動の分析

## 検証チェックポイント

送信後のエクスペリエンスを設定した後：

**設定の検証：**

- 選択した「ありがとうございます」オプションがフォームプロパティで正しく表示されるようになりました
- メッセージコンテンツがプレビューモードで正しく表示される
- リダイレクト URL が正しくフォーマットされ、アクセス可能である
- メッセージ内のすべてのリンクは正しく機能します

**ユーザーエクスペリエンステスト：**

- テストフォームを送信して、「ありがとうございます」メッセージが正しく表示されていることを確認します
- 異なるブラウザー間でのリダイレクト機能のテスト
- 「ありがとうございます」メッセージのモバイル応答の確認
- リダイレクトの宛先が正しく読み込まれていることを確認

**Analytics の設定：**

- 「ありがとうございます」メッセージ用に適切に実装されたトラッキングコード
- リダイレクト先のトラッキングが設定されました
- 目標完了イベントが正しく実行される

## 次の手順

送信後のエクスペリエンスの設定が成功したら、以下を行います。

- **パフォーマンスの監視**：分析を確認して、ありがとうメッセージに対するユーザーエンゲージメントを把握したり、ページをリダイレクトしたりします
- **反復と改善**：ユーザーのフィードバックとデータインサイトを使用して、送信後戦略を絞り込みます
- **スケール実装**：組織の他のフォーム全体で成功するパターンを適用する
