# ユニバーサルエディターを使用したレスポンシブFormsの作成

現代の Web 環境では、拡大し続けるデバイスや画面サイズ全体でシームレスに機能するフォームが必要です。 大きなデスクトップモニターからコンパクトなスマートフォン画面まで、ユーザーは選択したデバイスに関係なく、一貫性のある直感的なエクスペリエンスを期待しています。 レスポンシブフォームの作成はオプションではなくなりました。レスポンシブフォームは、コンバージョンに最適化され、アクセス可能でプロフェッショナルなデジタルエクスペリエンスを提供するための基本的な要件です。

ユニバーサルエディターは、様々な画面のサイズ、入力方法、ユーザーコンテキストにインテリジェントに適応するレスポンシブフォームを開発するための包括的なツールと手法を提供します。 このガイドでは、操作性、アクセシビリティ、視覚的魅力を維持しながら、すべてのデバイスで卓越したパフォーマンスを発揮するフォームを作成するために必要な技術的基礎、実装戦略、最適化手法について説明します。

レスポンシブフォームの作成には、次の 2 つの主なアクティビティが含まれます。

- **レスポンシブテスト：** デバイスエミュレーターを使用して、様々な画面サイズでフォームをプレビューおよびテストします。
- **レスポンシブデザイン：** 様々なデバイスにシームレスに適応するレイアウトパターンを選択し、実装します。

**このガイドでは、次の方法を説明します。**

- デスクトップ、タブレット、モバイルデバイスでのフォームのテスト
- コンテンツに適したレイアウトパターンの選択
- レスポンシブデザインのベストプラクティスの適用
- レスポンシブフォームに関するよくある問題のトラブルシューティング
- モバイルパフォーマンス向けにフォームを最適化

## レスポンシブFormsが重要な理由

**ユーザーエクスペリエンスへの影響：**

- 60% を超えるユーザーがモバイルデバイス上のフォームにアクセスしています
- モバイルエクスペリエンスが低いと、放棄率が 67% 高くなります
- レスポンシブフォームにより、完了率を最大 25% 向上させることができます

**ビジネス上のメリット：**

- フォームの完了率の上昇
- ユーザー満足度の向上
- アクセシビリティコンプライアンスの強化
- 開発コストとメンテナンスコストの削減

>[!TIP]
>
> **モバイルファーストのアプローチ：** モバイルデバイスの設計を開始してから、より大きな画面の拡張を行います。 これにより、コア機能が最も制約の厳しい環境で機能するようになります。

## 第 1 部：デバイス間でのFormsのテスト

異なるデバイス上のフォームをテストすると、ユーザーが問題に遭遇する前に、レスポンシブな問題を特定して解決するのに役立ちます。 ユニバーサルエディターは、様々な画面サイズと向きをシミュレートするエミュレーターモードを提供します。

### Formsのテスト方法

**手順 1：デバイスエミュレーターを開く**

1. フォームをユニバーサルエディターで開きます。
2. ツールバーの ![ エミュレーターアイコン ](/help/edge/docs/forms/universal-editor/assets/emulator.png){height=2%,width=2%} **エミュレーター** アイコンをクリックします。
3. デバイスセレクターメニューが表示されます。

![ ユニバーサルエディターのレスポンシブテストインターフェイス ](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**手順 2：テストするデバイスの選択**

- **デスクトップ** （1200px+幅）：デフォルトの編集ビュー
- **Tablet** （幅 768px-1199px）: Mediumのスクリーンテスト
- **モバイル** （幅 320px-767px）：小型画面のテスト
- **カスタム**：特定のデバイスの正確な寸法を指定します

**手順 3：デバイスの向きをテストする**

**スクリーンローテータ** アイコンを使用して、次の両方をテストします。

- **縦長モード**：標準のモバイル向き
- **横置きモード**：タブレットまたは電話の回転表示

### 医療機器テスト結果

各デバイスタイプには、一意のレスポンシブ動作が表示されます。

| **デバイスタイプ** | **画面の幅** | **確認対象** | **よくある問題** |
|-----------------|------------------|-------------------|-------------------|
| **デスクトップ** | 1200px+ | フルレイアウト、すべての機能を表示 | 過度の空白、過度に広いレイアウト |
| **タブレット** | 768px-1199px | コンポーネントの積み重ね、ナビゲーションの操作性 | サイズ設定が難しく、ターゲットに影響する問題が発生する |
| **モバイル** | 320px-767px | 1 列レイアウト、サムナビゲーション | 小さいテキスト、間隔の狭いボタン |
| **カスタム** | ユーザー定義 | デバイス固有の要件 | ブレークポイントエッジケース |

### ワークフローのテスト

**ニューFormsの場合：**

1. **デスクトップビューでの作成：** すべての機能を使って作業を開始します。
2. **タブレットでテスト：** 中程度の画面の適応を確認します。
3. **モバイルで検証：** 小さな画面での操作性を確保します。
4. **レスポンシブな問題の修正：** 必要に応じてレイアウトを調整します。
5. **すべてのデバイスを再テスト：** すべてのサイズで修正を確認します。

**既存のFormsの場合：**

1. **クイックモバイルチェック：** フォームは電話で機能しますか？
2. **問題領域の特定：** レイアウトと操作性の問題に注意してください。
3. **システマティックテスト：** 各デバイスのサイズを徹底的にテストします。
4. **ドキュメントのイシュー：** 修正が必要な事項を追跡します。
5. **修正の実装：** 問題に適切な方法で対処します。

## パート 2：レスポンシブレイアウトパターンの選択

レイアウトパターンは、様々な画面サイズに対するフォームコンテンツの適応方法を決定します。 適切なパターンにより、ユーザーエクスペリエンスとモバイルパフォーマンスの両方が向上します。

### レイアウトパターンの概要

| **レイアウトタイプ** | **次に最適** | **モバイルのパフォーマンス** | **複雑性** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **パネルレイアウト** | 分類されたコンテンツ、ダッシュボードスタイルのフォーム | 良好 | 低 |
| **ウィザードレイアウト** | 複数のステップから成るプロセス、複雑なワークフロー | 素晴らしい | 中 |
| **アコーディオンレイアウト** | FAQ スタイルのコンテンツ、オプションセクション | 素晴らしい | 低 |

### クイック決定ガイド

**パネルレイアウトを使用する場合：**

- コンテンツは異なるカテゴリに分類されています
- ユーザーは一度に複数のセクションを表示する必要がある
- コンテンツは比較的シンプルです

**次の場合にウィザード レイアウトを使用する：**

- フォームには複数の論理ステップがあります
- 認知負荷を軽減する
- モバイルユーザーは主なオーディエンスです

**アコーディオンレイアウトは次の場合に使用します。**

- フォームには、オプションまたはセカンダリコンテンツが含まれています
- 省スペースも重要
- コンテンツは論理的にグループ化できます

### パネルレイアウト

パネルレイアウトでは、関連するコンテンツが視覚的に異なるセクションに整理されるので、ユーザーは複数のセクションを一度に表示できます。 このレイアウトは、情報が分類されたフォームに最適です。大きな画面で横並びのプレゼンテーションを行うことができます。

![ パネルレイアウトの例 ](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**レスポンシブ動作**

- **デスクトップ（1200 px 以上）:** パネルが横に並んだり、グリッドに表示されたりして視認性が向上しています。
- **タブレット（768px-1199px）:** パネルが上下に適切な間隔で積み重ねられ、見やすさが維持されます。
- **モバイル （320px-767px）:** パネルは 1 列のレイアウトで表示され、セクション間を明確に分離してナビゲーションを容易にします。

**実装方法**

1. [ パネルコンポーネント ](https://experienceleague.adobe.com/ja/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel) をフォームに追加します。
2. 各パネル内の関連フィールドをグループ化して、論理的な組織を維持します。
3. 各パネルセクションに明確で説明的な見出しを割り当てます。
4. 見た目が乱雑になることを防ぐため、パネル間に十分なスペースを確保します。

**ベストプラクティス**

- デスクトップのパネル数を 3 個または 4 個に制限して、ユーザーが圧倒されないようにします。
- ユーザーの理解に役立つように、各パネルに簡潔で説明的なタイトルを使用します。
- パネル内のフィールドを論理的に整理して、認知的な負荷を最小限に抑えます。
- タッチデバイスでのパネルナビゲーションをテストして、すべてのプラットフォームでの操作性を確保します。

**一般的なユースケース**

- **求人：** 個人情報、教育、経験、参考資料のセクション。
- **製品登録：基本情報、技術仕様、保証情報の** パネル。
- **調査Forms:** 人口統計、環境設定、フィードバックおよび連絡先情報のグループ化。

### ウィザードレイアウト

ウィザードレイアウトでは、ユーザーは複数の手順に従ってプロセスを進め、1 つのセクションを一度に表示できます。 このレイアウトは、認識負荷を軽減し、プロセスを管理可能なステップに分割することで完了率を高めるので、複雑なフォームで特に効果的です。

![ ウィザードレイアウトの例 ](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**レスポンシブ動作**

- **すべてのデバイス：** モバイルユーザーに最適な、1 段階のフォーカスを維持します。
- **ステップコンテンツ：** 各ステップはレスポンシブに適応し、フィールドを積み重ねたり、画面サイズに応じてフィールドを並べて配置したりします。
- **ナビゲーション：** 適切な間隔を持つタッチフレンドリーなボタンを備えており、操作が簡単です。
- **進捗インジケーター：** 進捗バーまたはステップインジケーターは、異なるデバイスに対して適切にスケールされ、完了ステータスに関する明確なフィードバックを提供します。

**実装方法**

1. [ ウィザードコンポーネント ](https://experienceleague.adobe.com/ja/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard) をフォームに挿入します。
2. フォームを論理的な手順（理想的には 3～7 個）に分割して、各手順に集中して管理しやすくします。
3. 進行状況インジケーターを追加して、ユーザーがプロセス内での自分の位置を理解できるようにします。
4. 「次へ」、「戻る」、「保存」などの明確なナビゲーションコントロールを提供します。

**モバイルの最適化のヒント**

- アクセシビリティを向上させるには、ナビゲーションコントロールに大きなタッチターゲット（高さが 44 ピクセル以上）を使用します。
- 小さい画面でステップインジケーターが表示され、読みやすいことを確認します。
- 1 ステップあたりのフィールド数を制限して、スクロールを最小限に抑え、フォーカスを改善します。
- ユーザーがフォームを離れた場合にデータが失われないように、自動保存機能を有効にします。

**ベストプラクティス**

- 論理的な進行に従う手順を設計し、各手順を前の手順に基づいて構築します。
- 各手順にはわかりやすいタイトルを使用し、ユーザーの期待事項を設定します。
- 各ステップでユーザー入力を検証して、エラーを早期に発見し、フラストレーションを軽減します。
- ユーザーは、データを失うことなく過去の情報を確認または編集するために後方に移動できます。

**一般的なユースケース**

- **保険金請求：インシデントの詳細、証拠の提出、個人情報および確認の** 手順。
- **アカウント設定：基本情報、環境設定、セキュリティ設定、確認の** 段階。
- **注文プロセス：商品選択、発送情報、支払い詳細、注文概要の** つのステップがあります。

### アコーディオンレイアウト

アコーディオンレイアウトは、コンテンツを折りたたみ可能なセクションに整理することでスペースを節約し、オプションの情報やセカンダリ情報に最適です。 このレイアウトは、論理的にグループ化でき、一度にすべてを表示する必要がないコンテンツを含むフォームで特に効果的です。

![ アコーディオンレイアウトの例 ](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**レスポンシブ動作**

- **モバイルのパフォーマンス：** 関連するセクションのみが展開され、スクロールの必要性が減り、読み込み時間が短縮されます。
- **タッチ操作向けヘッダー：** セクションヘッダーは、タップや展開が簡単で、モバイルデバイスでの自然なジェスチャーをサポートします。
- **スムーズ アニメーション：** セクションを展開したり折りたたんだりすると、ユーザの操作に関する視覚的なフィードバックが得られます。
- **スペースの効率性：** 折りたたまれたセクションは、垂直方向のスペースを最小限に抑え、すべてのデバイス上でフォームを簡単に移動できるようにします。

**実装方法**

1. フォームに [ アコーディオンコンポーネント ](https://experienceleague.adobe.com/ja/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion) を追加します。
2. 各アコーディオンセクション内の関連するオプションコンテンツまたはセカンダリコンテンツをグループ化します。
3. 各セクションにはわかりやすいヘッダーを使用し、どのような情報が含まれているかをユーザーが理解できるようにします。
4. 重要度とユーザーのニーズに基づいて、各セクションに適切なデフォルトのオープン状態またはクローズ状態を設定します。

**モバイルのメリット**

- 使用されていないセクションを折りたたむことでスクロールを減らし、ユーザーが一度に 1 つのセクションに集中できるようにします。
- タッチ操作に対応し、自然な展開/折りたたみジェスチャーに対応。
- アクティブコンテンツのみが表示されるので、読み込みが高速になります。
- ユーザーは必要な情報のみを常に表示できるため、焦点が絞られるようになりました。

**ベストプラクティス**

- セクションを展開する前にユーザーが何を期待するかを把握できるように、セクション ヘッダーをクリアを使用します。
- 関連するコンテンツを各セクション内で論理的にグループ化して、理解を助けます。
- 緊急の注意が必要な場合は、重要なセクションを展開して開始します。
- ユーザーが拡張するセクションを決定するのに役立つ、簡単なセクションのプレビューまたは要約を提供します。

**一般的なユースケース**

- **製品の設定：基本オプション、詳細設定、アクセサリ、サポートの** セクション
- **よくある質問（FAQ） Forms：アカウント、請求、技術、一般に関する質問のグループ化を** います。
- **設定Forms：プライバシー、通知、外観、および詳細オプションに関する** セクション。


## 第 3 部：レスポンシブデザインのベストプラクティス

### デバイスタイプ別のベストプラクティス

+++モバイルの最適化（320px-767px）

**レイアウトとインタラクション：**

- すべてのフォームコンテンツに対して 1 列レイアウトを使用して、読みやすさと使いやすさを最大限に高めます。
- 信頼性の高いタッチインタラクションを行うために、すべてのボタンとインタラクティブ要素の高さが 44 px 以上あることを確認します。
- 「戻る」ボタンと「次へ」ボタンが表示され、明確でシンプルなナビゲーションを提供します。
- 長いフォームを分割して、各セクション内でスクロールする必要性を最小限に抑えます。
- 最初の入力フィールドに自動的にフォーカスして、モバイルキーボードにプロンプトを表示します。

**フィールドのガイドライン：**

- テキストフィールドは、タッチ入力に十分なパディングを使用して、画面の全幅に広がる必要があります。
- 最適なモバイルのユーザビリティを得るには、ネイティブのドロップダウン/選択要素を使用します。
- ネイティブの日付選択を実装して、一貫したモバイルエクスペリエンスを実現します。
- ファイルのアップロードエリアを大きくし、明確にラベル付けしてアクセスしやすくします。

+++

+++タブレットの最適化（768px-1199px）

**レイアウトと操作性：**

- 関連するフィールドに 2 列のレイアウトを利用して、画面スペースの増加を活用します。
- 縦向きと横向きの両方でフォームの外観と操作性をテストします。
- タッチ入力とマウス入力の両方に対応したデザインで、すべてのコントロールに簡単にアクセスできます。
- 明確な視覚階層と読みやすさを維持しながら、コンテンツ領域のサイズを大きくします。

+++

+++デスクトップの最適化（1200px+）

**高度な機能とレイアウト：**

## トラブルシューティング

### レイアウトの問題

+++モバイルのフォームレイアウトが壊れる

**考えられる原因：**

- 小さい画面に適応しない固定幅要素
- モバイルスタイルをオーバーライドするデスクトップファーストの CSS
- コンテナがあふれている画像またはコンテンツ

**修正方法：**

- すべての画像とコンテナで、相対サイズまたはパーセンテージベースのサイズ設定を使用していることを確認します。
- モバイルファーストの CSS アプローチから始めて、より大きな画面に対する機能強化を重点的に説明します。
- デバイスエミュレーターと実際のデバイスの両方を使用してフォームをテストします。
- 固定寸法は避け、柔軟なレイアウトを使用します。

+++

+++タッチターゲットが小さすぎる

**考えられる原因：**

- 44 ピクセル x 44 ピクセル未満のボタンまたはリンク
- インタラクティブな要素が近すぎる
- デフォルトのタッチターゲットサイズを縮小するカスタム CSS

**修正方法：**

- すべてのインタラクティブ要素が 44 ピクセル x 44 ピクセル以上であることを確認します。
- ボタン、リンク、その他のコントロールの間に適切な間隔を追加します。
- マウスだけでなく、実際のタッチデバイスでテストします。
- 必要に応じて、アクセシビリティに関するタッチターゲット領域を拡大します。

+++

+++コンテンツオーバーフローの問題

**考えられる原因：**

- 長いテキストまたは折り返さないラベル
- 幅が固定されたコンテナ
- レスポンシブに拡大・縮小されない画像

**修正方法：**

- すべてのラベルとコンテンツでテキストの折り返しを有効にします。
- コンテナと共に拡大・縮小するレスポンシブ画像を使用します。
- 様々なコンテンツの長さに適応する柔軟なレイアウトを設計します。
- 短いコンテンツと長いコンテンツの両方でテストして、適応性を確保します。

+++

### パフォーマンスの問題

+++モバイルでの読み込みが遅い

**考えられる原因：**

- 最適化されていない大きな画像
- 重い、または過剰なJavaScript
- 同時に読み込むフォームフィールドが多すぎます

**修正方法：**

- モバイル用に画像を最適化し、適切なファイル形式を使用します。
- 重要でないコンテンツを遅延または遅延読み込みする。
- サードパーティのスクリプトおよびウィジェットの使用を最小限に抑える。
- フォームフィールドを効率化して、必要なもののみを読み込みます。

+++

### テストと検証の問題

+++エミュレーターと実際のデバイスの違い

**考えられる原因：**

- ブラウザーのレンダリングエンジンの違い
- マウスで正確にシミュレーションできないタッチ操作
- ネットワーク速度の不一致

**修正方法：**

- エミュレーターに加えて、常に実際のデバイスでテストします。
- 包括的なテストを行うには、複数のブラウザーとデバイスを使用します。
- 様々なネットワーク速度をシミュレートして、パフォーマンスのボトルネックを特定します。
- ターゲットオーディエンスの実際のユーザーからフィードバックを収集します。

## 次の手順

**即時アクション：**

1. **現在のフォームを監査：** デバイスエミュレーターを使用して既存のフォームをテストします。
2. **簡単な勝利の特定：** まず、モバイルの操作性に関する明白な問題を修正します。
3. **トラフィック量の多いフォームに優先順位を付ける：** ユーザーへの影響が最も大きいフォームに焦点を当てます。
4. **モバイルファーストのアプローチを実装：** 最小の画面デザインから始めます。

**高度な最適化：**

- **パフォーマンス監視：** フォーム指標を追跡する Analytics を設定します。
- **A/B テスト：** 様々なレイアウトやアプローチで実験します。
- **ユーザーフィードバックの収集：** 実際のユーザーからインサイトを収集します。
- **継続的な改善：** フォームを定期的に確認し、最適化します。
