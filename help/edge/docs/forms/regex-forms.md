# 検証で一般的に使用される正規表現の式

以下に、最新のブラウザーによる機能以上に、フォームの検証機能を強化するために使用できる正規表現を示します。

## 堅牢なパスワード

```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$
```

次の文字を含む 8 文字以上を確認します。

* 小文字 (a ～ z)
* 大文字 (A ～ Z)
* 数値 (0 ～ 9)
* 特殊文字 (@$!%*?&amp;)


## メールアドレス


```regex
^[a-zA-Z0-9.!#$%&'*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$
```

ユーザ名とドメイン名に文字、数字、特殊文字を使用できます。


## 電話番号（米国形式）

```regex
^\(?([0-9]{3})\)?[-. ]([0-9]{3})[-. ]([0-9]{4})$
```

形式 (XXX)XXX-XXXX の電話番号を検証します。



## URL

```regex
^(http|https)://.*$
```

http または https で始まる有効な URL を指定します。



## 日付 (YYYY-MM-DD)

```regex
^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12][0-9]|3[01])$
```

YYYY-MM-DD 形式の日付を検証します。


## 時刻 (HH:MM)

```regex
^([01][0-9]|2[0-3]):[0-5][0-9]$
```

HH:MM（24 時間形式）の時刻を検証します。


## 郵便番号（米国形式）

```regex
^\d{5}(?:[-\ ]\d{4})?$
```

5 桁の米国の郵便番号をオプションのハイフンと 4 桁の拡張子で検証します。


## ユーザー名（英数字およびアンダースコア）

```regex
^[a-zA-Z0-9_]+$
```

文字、数字、アンダースコアを使用できます。


## カラー 16 進コード

```regex
^#[0-9a-fA-F]{6}$
```

6 桁の 16 進数の色コードを検証します。 例：#FFFFFF.


## IP アドレス

```regex
^(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})$
```

IPv4 アドレスを検証します。



## 社会保障番号（米国形式）

```regex
^\d{3}-\d{2}-\d{4}$
```



## クレジットカード番号

```regex
^(?:4[0-9]{12}(?:[0-9]{3})?|[25][1-7][0-9]{14}$
```

形式 (XXX)XXX-XXXX の電話番号を検証します。



## 電話番号（米国形式）:

```regex
^[a-zA-Z0-9.!#$%&'*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$
```

形式 (XXX)XXX-XXXX の電話番号を検証します。
