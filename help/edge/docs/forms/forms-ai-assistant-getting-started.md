# Forms Experience Builder の基本を学ぶ

>[!NOTE]
>
> Forms Experience Builder 機能は、**早期アクセス（EA）プログラム** で利用できます。 勤務先のアドレスから `aem-forms-ea@adobe.com` にメールを送信して、機能へのアクセスをリクエストします。

>[!IMPORTANT]
>
> **ドキュメントは変更される場合があります**：このドキュメントは現在製品に対してテスト中であり、更新および改訂される可能性があります。Forms Experience Builder が早期アクセスプログラム中に進化し続けると、機能、コマンド、例が変わる場合があります。

この包括的なガイドは、対話型 AI テクノロジーを使用したフォームの作成と管理を開始する際に役立ちます。初めてフォームを作成する初心者でも、高度な機能を活用する上級ユーザーでも、Forms Experience Builder の機能に関する詳細な情報と実用的な例を見つけることができます。

## 前提条件と設定

### &#x200B;1. アクセスをリクエスト

Forms Experience Builder は現在、早期アクセス（EA）プログラムの一部として利用できます。参加してアクセスするには、次の情報が必要です：

**必要な情報**

- **IMS 組織 ID**:Adobeの組織識別子
- **プログラム ID**:Adobe Experience Cloud内の固有のプログラム ID
- **プロジェクト詳細**：タイムライン、範囲、意図されたユースケース
- **仕事用の公式メール**：組織のAdobe アカウントに関連付けられています

**IMS 組織 ID とプログラム ID の取得方法**

IMS 組織 ID とプログラム ID を見つける詳細な手順については、以下を参照してください。

- [Adobe Experience Cloud組織セットアップガイド](/help/onboarding/cloud-manager-introduction.md)
- [プログラムと環境の管理](/help/implementing/cloud-manager/getting-access-to-aem-in-cloud/program-types.md)

**アクセスを要求**

1. 上記のガイドを使用して、IMS 組織 ID とプログラム ID を収集します
2. [aem-forms-ea@adobe.com](mailto:aem-forms-ea@adobe.com) にアクセスを要求するメールを送信
3. リクエストに含める：
   - 組織名と IMS 組織 ID
   - プログラム ID
   - プロジェクトのタイムラインと範囲
   - 使用目的と事業目的

>[!IMPORTANT]
>
> **限定提供プログラム**:Forms Experience Builder へのアクセスは、内部の関係者の承認が必要です。 Adobeは、プログラムの処理能力と早期アクセス基準との整合性に基づいて、お客様のリクエストを確認します。 承認は保証されず、現在のプログラムの可用性によって異なります。

### &#x200B;2. フォームが有効になっていることを確認

Forms Experience Builder を使用する前に、[お使いの環境で AEM Forms が有効になっていること](/help/forms/setup-forms-cloud-service.md)を確認します。


### &#x200B;3. 環境を設定


- **Edge Delivery Services（EDS）の場合：**

   - [Edge Delivery Services フォームの環境を設定します](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)
   - [Edge Delivery Forms テンプレートを使用して新規フォームを作成します](/help/edge/docs/forms/universal-editor/create-forms.md)

- **コアコンポーネントベースのフォームの場合：**

   - Adobe Experience Manager インスタンスで、Forms／フォームとドキュメントに移動します
   - [コアコンポーネントテンプレートを使用して新規ページを作成します。](/help/forms/creating-adaptive-form-core-components.md)


## クイックスタート

### Forms Experience Builder へのアクセス

Forms Experience Builder は、Forms管理 UI、ユニバーサルエディター、アダプティブ Forms エディターで利用できます。 これらのメソッドのいずれかを使用して、フォームにアクセスできます。

**Forms Management UI （コアコンポーネント用）**

1. **Formsに移動**: AEM/Forms/Formsとドキュメントに移動します。
1. ツールバーのForms Experience Builder アイコンをクリックします。 UI の左上付近にあります。
   ![AI アシスタントアイコン*](/help/edge/docs/forms/assets/forms-manager.gif){width="50%"}
1. 対話型フォームの作成を開始する


**アダプティブFormsエディター（コアコンポーネント用）**

1. AEM/Forms/Formsとドキュメントに移動します。
2. [コアコンポーネントテンプレートを使用して新しいフォームを作成](/help/forms/creating-adaptive-form-core-components.md)
3. フォームを編集用に開きます
4. エディターツールバーの Forms Experience Builder アイコンをクリックします
   ![AI アシスタントアイコン*](/help/edge/docs/forms/assets/adaptive-forms-editor.gif){width="75%"}

5. 対話型フォームの作成を開始する


**ユニバーサルエディター（Edge Deilvery Services Forms用）**

1. [Edge Delivery Services設定ガイドに従って ](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)EDS ページを作成します
1. ユニバーサルエディターで EDS ページに移動
1. 右側のパネルに表示されるForms Experience Builder アイコンを見つけます
1. クリックして、対話型インターフェイスを開きます。

### 必須コマンド

| シンボル | 目的 | 使用例 |
|--------|---------|---------------|
| `/` | クイックアクションとショートカット | `/create-form contact form`、`/help validation rules`、`/update-layout wizard` |
| `@` | 既存のフォームフィールドを参照 | `@email`、`@firstName`、`Make @phoneNumber required` |
| プレーンテキスト | 自然な対話 | 「必須の電話番号フィールドを追加」、「メールの検証を作成」 |

**特定のコマンドの例：**

- `/create-form customer survey` – 新規顧客調査フォームを作成します
- `/add-field @email validation` – 既存のメールフィールドに検証を追加します
- `/create-rule show @spouse if @maritalStatus equals married` – 条件ロジックを作成します
- `/configure-submit to email support@company.com` - メール送信の設定
- `/help multi-step forms` – 複数ステップのフォーム作成に関するヘルプを取得します

### 成功に向けたヒント

- **具体的に指定**：「検証付きの必須メールフィールドを追加してください」は、「メールを追加してください」よりも効果的です
- **既存のフィールドを参照**：フォームを変更する際に `@fieldName` を使用します
- **ヘルプを求める**： `/help` に続けて質問を入力します
- **反復**：最良の結果を得るには、一度に 1 つずつ変更を行います


## フォームの作成を開始する方法

### &#x200B;1. 自然言語プロンプトから開始

フォームの要件を自然言語で記述すると、Forms Experience Builder により、完全なフォーム構造が生成されます。

**例：**

- 「個人情報、財務詳細、ドキュメントアップロードを含むローン申請フォームを作成してください」
- 「評価、コメント、製品カテゴリを含む顧客フィードバックフォームを作成してください」
- 「支払い処理を伴う会議には複数のステップから成る登録フォームが必要です」

### &#x200B;2. 読み込みと変換

既存のフォームやドキュメントを最新のインタラクティブなエクスペリエンスに変換します。

**サポートされているソース：**

- **PDF フォーム**：静的 PDF をアップロードして、検証機能を備えたインタラクティブデジタルフォームに変換します。
- **スクリーンショットまたは画像**：紙のフォームの写真をアップロードして、機能的なデジタルバージョンを生成します
- **XFA フォーム**：従来の XFA ベースのフォームを最新のレスポンシブフォームに変換します

**読み込み方法：**

1. Forms Experience Builder インターフェイスの添付ファイルアイコンをクリックします
2. ファイル（PDF、画像、Figma デザインなど）をアップロードします
3. 要件を説明します。
   - 「この PDF フォームをデジタル版に変換してください」
   - 「このスクリーンショットレイアウトに一致するフォームを作成してください」
   - 「Figma デザインからこのフォームを作成してください」

**サポートされているファイルタイプ：**

- **画像** （PNG、JPG、GIF）：フォームレイアウト、UI モックアップ、スキャンされたフォーム、手描きのスケッチ
- **PDF ファイル**：既存のフォーム、仕様、ドキュメント、Acroforms、XFA フォーム
- **スクリーンショット**：デスクトップ/モバイルアプリのスクリーンショット、紙のフォームの写真、ホワイトボードのスケッチ
- **手描きスケッチ**：ナプキンスケッチ、ワイヤーフレーム、コンセプトドローイング（写真）

## ベストプラクティス

### フォームデザイン

- **シンプルに保持**：必須フィールドから開始し、ユーザーに負担をかけないように、必要な場合にのみ複雑さを追加します
- **明確なラベルを使用**：フォームを通じてユーザーをガイドする説明ラベルを使用して、フィールドの目的を明確にします
- **ヘルプテキストを提供**：コンテキストヘルプと例を使用して、複雑なフィールドについてユーザーをガイドします
- **徹底的にテスト**：すべてのユーザーパスを検証して、すべてのシナリオでフォームが正しく機能することを確認します

### ユーザーエクスペリエンス

- **プログレッシブ開示**：コンテキストに基づいて関連フィールドを表示し、認知的負荷を軽減して完了率を向上させます
- **明確なナビゲーション**：ユーザーがフォーム内の位置と、残りの手順を理解できるようにします
- **レスポンシブデザイン**：アクセシビリティを最大限に高めるために、フォームがすべてのデバイスと画面サイズで機能することを確認します
- **アクセシビリティ**：WCAG ガイドラインに従って、障がい者がフォームを使用できるようにします

### パフォーマンス

- **フィールド数を最適化**：フォームの放棄を減少させ、完了率を向上させるために必要な情報のみを要求します
- **適切な検証を使用**：送信前にエラーを防ぎ、即時のフィードバックとガイダンスを提供します
- **テスト完了率**：分析とユーザーからのフィードバックを通じてフォームの有効性を監視および改善します
- **定期的な更新**：最適なパフォーマンスを実現するために、ビジネスニーズとユーザーの期待に合わせてフォームを最新の状態に保持します

### ブランドの一貫性

- **ブランドテンプレートを作成**：フォームの作成を開始する前に、組織のカラー、フォント、スタイルを使用してブランド化されたフォームテンプレートを準備します
- **スタイル標準を定義**：プロンプトで参照できる一貫性のあるボタンスタイル、フィールドレイアウト、間隔ガイドラインを確立します
- **ブランドアセットを使用**：フォームを作成する際に簡単に参照できるように、ロゴ、カラーコード、ブランドガイドラインを準備します
- **テンプレートライブラリ**：一般的なユースケース（連絡先、登録、フィードバック）向けのブランド化されたフォームテンプレートのコレクションを作成します
- **スタイルプロンプト**：「ボタンには会社の青（#1234AB）を使用し、企業フォントは Helvetica を使用します」のようなブランド固有の手順を含めます



## トラブルシューティング

| 問題 | 簡単な修正 |
|-------|-----------|
| **インターフェイスが読み込まれない** | ブラウザーを更新し、インターネット接続を確認します |
| **コマンドが機能しない** | `/help` を試すか、代わりに自然言語を使用します |
| **@fieldName が認識されない** | スペルをチェックし、最初にフィールドが存在することを確認します |
| **ファイルのアップロードに失敗する** | 10 MB 未満の PDF／JPG／PNG を使用します |
| **フォームが正しく表示されない** | より具体的に、「モバイル対応にしてください」を指定します |
| **設定の送信に失敗** | API 資格情報と権限を検証します |

**まだヘルプが必要ですか？**`/help` に続いて具体的な質問を入力するか、システム管理者にお問い合わせください。
