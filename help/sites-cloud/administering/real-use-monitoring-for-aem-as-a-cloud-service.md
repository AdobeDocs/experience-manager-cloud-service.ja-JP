# Cloud ServiceとしてのAEMのための実使用監視サービス {#real-use-monitoring-service-for-aem-as-a-cloud-service}

>[!NOTE]
>
>データのクライアントサイド コレクションである実使用監視サービスは、自動化されたサービスです。 顧客のセットアップは必要ありません。

>[!INFO]
>
>クライアント側の監視はAEM Cloud Serviceバージョン **2024.5.16461** 以降を使用しているお客様に対してのみ機能します。

## 概要 {#overview}

リアルユースモニタリング(RUM)サービスは、Webサイトまたはアプリケーションのデジタルユーザーエクスペリエンスをリアルタイムでキャプチャして分析するパフォーマンス監視テクノロジーです。 これにより、Web アプリケーションのリアルタイムパフォーマンスを可視化し、エンドユーザーエクスペリエンスをより深くインサイトできます。 このサービスは、ユーザー自身ではなく、Webサイトのエンゲージメントを監視することにより、パフォーマンスを最適化することに重点を置いています。

RUM を使用すると、URLの開始からリクエストがブラウザーに返送されるまで、主要パフォーマンス指標が追跡されます。 これによりデベロッパーアプリケーションが強化され、エンド ユーザーにとって使いやすくなります。

>[!INFO]
>
>「リアルユーザーモニタリング」は、サービスの真の本質をよりよく反映しているため、「リアルユースモニタリング」にブランド名が変更されました。

## 実際の監視サービスの恩恵を受けることができるのは誰ですか? {#who-can-benefit-from-rum-service}

実使用監視サービスは、すべてのお客様にとって有益です。 これは、ユーザーインタラクションの代表的な反映を提供し、クライアントサイド ページビューの数をキャプチャすることにより、Webサイトエンゲージメントの信頼できる測定するを保証します。

このサービスは、Adobe Systemsすべてのお客様にとって、ユーザーインタラクションに関する貴重な洞察を提供します。 独自のCDNを使用しているお客様は、Adobe Systemsデータ収集を直接統合し、更新サイクル中に個別のレポートを作成する必要がなくなるため、簡素化されたトラフィック レポートの恩恵を受けることができます。

## 実際の使用監視サービスのしくみを理解する {#understand-how-the-rum-service-works}

Adobe Experience Manager(AEM)は、リアルユースモニタリング(RUM)を使用して、訪問者がAEMサイトとどのようにやり取りするかを顧客とAdobe Systems理解できるようにします。 パフォーマンスの問題を診断し、実験の有効性測定するのに役立ちます。 RUM はサンプリングを通じて訪問者のプライバシーを保護し(ページビュー全体のごく一部のみが監視されます)、個人を特定できる情報(PII)は収集されません。

## 実使用の監視サービスとプライバシー {#rum-service-and-privacy}

AEM の実使用監視サービスは、訪問者プライバシーを保護し、データ収集を最小限に抑えるように設計されています。 訪問者として、それはあなたが訪問している、またはAdobe Systemsに利用可能になっているサイトが個人情報を収集しないことを意味します。

サイト運営者は、この機能による監視を有効にするために追加のオプトインは必要ありません。 エンドユーザーが RUM を有効にするために受け入れる追加のポップアップや同意書はありません。

## サンプリングサービス データ実使用の監視 {#rum-service-data-sampling}

従来の web 分析ソリューションでは、すべての訪問者に関するデータを収集しようとします。AEMのRUMサービスは、ページビューのごく一部からのみ情報をキャプチャします。 このサービスは、分析に代わるものではなく、サンプリングと匿名化を目的としています。 デフォルトでは、ページのサンプリング率は 1:100 です。 現時点では、サイト運営者はサンプリング率を増減できません。 合計トラフィックを正確に見積もるために、100 ページビューごとにデータが 1 から収集されるため、全体的なトラフィックの信頼できる近似値が得られます。

データを収集するかどうかの決定として、ページビューベースでページビュー行われ、複数のページにわたる相互作用を追跡することは事実上不可能になります。 設計上、RUM には訪問者やセッションの概念はなく、ページビューのみの概念があります。

## 収集されるデータ {#what-data-is-being-collected}

実使用監視サービスは、個人を特定できる情報コレクションを防止するように設計されています。 RUM によって収集される情報の完全なセットを以下に示します。

* 訪問しているサイトのホスト名（例：`experienceleague.adobe.com`）
* ページの表示に使用される幅広いユーザーエージェントの種類とオペレーティング システム。 `desktop:windows` または `mobile:ios`
* `2021-06-26 06:00:02.596000 UTC (in order to preserve privacy, we round all minutes to the previous hour, so that only seconds and milliseconds are tracked)` など、データ収集の時間
* 訪問しているページの URL（例：`https://experienceleague.adobe.com/docs`）
* リファラー URL（ユーザーがリンクをたどった場合、現在のページにリンクしているページの URL）
* `2Ac6` のような形式で、ランダムに生成されたページビューの ID
* `100` など、サンプリングレートの重み付けまたはその逆これは、100回のページビューに1回しか記録されないことを意味します
* ページの読み込みシーケンスにおける特定のイベントのチェックポイントまたは名前。 または、訪問者として操作します
* 上記のチェックポイントに対してユーザーが操作する DOM 要素のソースまたは識別子。 インスタンス、画像にすることができます
* ターゲット（上記のチェックポイントでユーザーがやり取りする外部ページまたはリソースへのリンク）。例：`https://blog.adobe.com/jp/publish/2022/06/29/media_162fb947c7219d0537cce36adf22315d64fb86e94.png`
* コアWebバイタル(CWV)のパフォーマンス指標には、最大のコンテンツフルペイント(LCP)、ファースト入力 ディレイ(FID)、累積レイアウトシフト(CLS)、時間から最初のバイト(TTFB)など、訪問者のエクスペリエンスの品質を表します。

## 顧客の実際の使用状況の監視のしくみ {#how-rum-works-for-a-customer}

リアルユースモニタリングは、クライアントサイド トラフィックを自動的に監視して、貴重な洞察を提供します。 Adobe Systems 顧客として、このサービスは既存のセットアップにシームレスに統合されているため、追加の手順を実行する必要はありません。 一般可用性 (GA) ロールアウトを使用すると、この新機能のメリットが自動的に享受できます。

## 実際の使用状況の監視サービス データの使用方法 {#how-rum-service-data-is-being-used}

RUM データは、次の目的で有益です。

* お客様のサイトに対してパフォーマンスのボトルネックを特定および修正する
* ページビューを含む自動トラフィックルックアップを効率化するため。
* 互換性を高めるために、AEMが同じページ上の他のスクリプト(分析、ターゲティング、外部ライブラリなど)とどのように相互作用するかを理解する。

## ページビューおよびパフォーマンス指標の制限事項と差異について {#limitations-and-understanding-variance-in-page-views-and-performance-metrics}

RUM データを分析すると、ページビューやその他のパフォーマンスメトリクスにばらつきが生じる場合があります。 これらの差異は、リアルタイムのクライアントサイドのモニタリングに固有のいくつかの要因に起因する可能性があります。お客様が RUM データを解釈する際に留意すべき重要な考慮事項を以下に示します。

1. **トラッカーブロッカー**

   * トラッカーブロッカーまたはプライバシー拡張機能を使用している終了ユーザーは、これらのツールがトラッキングスクリプトの実行を制限するため、RUM データ収集を妨げる可能性があります。 この制限は、過小報告されたページビューとユーザーインタラクションリード、実際のサイトアクティビティとRUMによってキャプチャされたデータとの間に不一致が生じる可能性があります。

1. **ヘッドレス API/JSON 呼び出しのキャプチャにおける制限**

   * RUM データサービスはクライアントサイド エクスペリエンスに重点を置いており、現時点では、AEM 以外のヘッドレスアプリから行われたバックエンド API や JSON 呼び出しをキャプチャしません。 RUM サービスデータからこれらのコールを除外すると、CDN Analyticsによって測定された内容リクエストとの差異が生じます。

## FAQ {#faq}


1. **顧客はRUMサービススクリプトをDynatraceなどのサードパーティシステムと統合できますか?**

   はい。

1. **「次のペイントへのインタラクション」、「最初のバイトへの時間」、「最初のコンテンツフルペイント」Webバイタル指標が収集されていますか?**

   次のペイントへの相互作用(INP)と最初のバイトへの時間(TTFB)が収集されます。  現時点では、コンテンツの初回ペイント（FCP）は収集しません。

1. **サイトで `/.rum` パスがブロックされていますが、どうすれば修正できますか?**

   RUM コレクションが機能するには、 `/.rum` パスが必要です。 Adobe Systems がCloud Serviceの一部として AEM として提供するものの前に CDN がある場合は、 `/.rum` パスが他のAEM 内容と同じAEM 接触チャネルに転送されるようにします。 また、調整されていないことを確認してください。

1. **RUMコレクション契約上の目的で内容要求にカウントされますか?**

   RUM ライブラリと RUM コレクションは内容リクエストとしてカウントされず、ページビューまたは API 呼び出しの報告数も増加しません。 さらに、AEM をCloud Serviceとして使用した標準の CDN を使用するお客様の場合、 [サーバーサイド コレクション](#serverside-collection) は内容要求の基礎となります。

1. **どうすればオプトアウトできますか?**

   Adobe Systems、リアルユースモニタリング(RUM)には大きなメリットがあり、デジタルエクスペリエンスを最適化できるため、使用することをお勧めします。 Web サイトのパフォーマンス向上に役立つ貴重なインサイトをオファーできます。 このサービスはシームレスに設計されており、Webサイトのパフォーマンスに影響を与えません。

   オプトアウトは、これらの洞察を見逃すことを意味します。 ただし、問題が発生した場合は、Adobe Systems サポートにお問い合わせください。
